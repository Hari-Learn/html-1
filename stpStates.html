Spanning Tree Protocol (STP) – Port States

What are STP Port States?

STP port states define how switch ports behave while preventing Layer 2 loops. Each port moves through different states before it can forward traffic.

1. Disabled

The Disabled state means the port is administratively shut down or not operational.

  • No traffic is sent or received
  • No MAC addresses are learned
  • No BPDUs are processed

2. Blocking

The Blocking state prevents loops in the network.

  • Does NOT forward data frames
  • Does NOT learn MAC addresses
  • Receives and processes BPDUs

3. Listening

In the Listening state, the port prepares to forward traffic.

  • Does NOT forward data frames
  • Does NOT learn MAC addresses
  • Sends and receives BPDUs

4. Learning

The Learning state allows the switch to build its MAC table.

  • Does NOT forward data frames
  • LEARNS MAC addresses
  • Sends and receives BPDUs

5. Forwarding

The Forwarding state is the normal operating state.

  • Forwards data frames
  • Learns MAC addresses
  • Sends and receives BPDUs

STP State Transition Flow

Blocking → Listening → Learning → Forwarding

Note: Disabled and Blocking ports do not forward traffic.
