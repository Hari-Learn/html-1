NAT (Network Address Translation)

1. What is NAT?

NAT is a process where a network device (usually a router) translates **private IP addresses** within a local network to a **public IP address** for communication over the internet, and vice versa. This allows multiple devices in a private network to share a single public IP.

2. NAT Translation Animation

The blue circle represents a private IP packet. When it passes through NAT, it changes to a public IP (red) and goes to the internet.

3. Types of NAT

  • Static NAT: One-to-one mapping of private to public IP.
  • Dynamic NAT: Private IP mapped to a public IP from a pool.
  • PAT (Port Address Translation): Many-to-one mapping using different ports (also called NAT overload).

4. NAT Conditions

When configuring NAT, at least **one ACL (Access Control List) or rule** is needed to define which source IP addresses can be translated. This ensures that only authorized traffic from the private network is allowed for NAT translation.

5. How NAT Works

Private to Public: A device with IP 192.168.1.2 sends a packet to the internet. NAT router replaces the source IP with a public IP.
Public to Private: Response from internet comes to the public IP. NAT router maps it back to 192.168.1.2 using the NAT table.
