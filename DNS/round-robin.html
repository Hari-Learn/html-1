DNS Routing Policy

Round Robin DNS

Round Robin DNS distributes incoming requests across multiple servers by rotating DNS responses in sequence. It is one of the simplest traffic distribution techniques used in DNS-based load balancing.

Routing Type
Traffic Distribution
Complexity
Low
Health Awareness
No

How Round Robin DNS Works

1

Multiple IP Records

A single domain name is configured with multiple A or AAAA records pointing to different servers.

2

DNS Rotation

The DNS server rotates the order of returned IP addresses for each incoming DNS query.

3

Traffic Distribution

Clients connect to different servers based on the IP address returned first by the resolver.

Example DNS Records 

example.com    A    192.168.1.10
example.com    A    192.168.1.11
example.com    A    192.168.1.12
Query #1
192.168.1.10
Query #2
192.168.1.11
Query #3
192.168.1.12

Advantages

  • ✅ Very easy to configure
  • ✅ Distributes traffic across servers
  • ✅ No additional load balancer required
  • ✅ Low infrastructure cost
  • ✅ Works globally using DNS

Limitations

  • ❌ No health checking by default
  • ❌ Uneven traffic distribution possible
  • ❌ DNS caching affects balancing
  • ❌ No session persistence
  • ❌ Not ideal for dynamic workloads
