Cloud DNS Routing Policies

Understanding Simple and Weighted Routing Strategies

What is DNS Routing Policy?

DNS routing policies determine how traffic is directed to cloud resources. Different routing methods help improve performance, availability, and traffic distribution across servers.

Simple Routing Policy

Simple routing sends 100% of the traffic to a single resource or endpoint.

Use Case

  • Single web server deployment
  • Basic applications
  • Low complexity infrastructure

Traffic Flow

Users
➡️
Single Server

All requests are routed to one destination.

Weighted Routing Policy

Weighted routing allows you to specify the percentage of requests sent to different resources.

Example Distribution

Server A 80%
Server B 20%

A/B Testing

Send small percentages of traffic to new application versions.

Load Distribution

Balance traffic across multiple cloud resources.

Migration Strategy

Gradually move users from old infrastructure to new systems.

Routing Policy Comparison

Feature Simple Routing Weighted Routing
Traffic Distribution 100% to one resource Distributed by percentage
Complexity Simple Moderate
Best For Single server apps Traffic balancing & testing
Traffic Control No customization Fine-grained control
