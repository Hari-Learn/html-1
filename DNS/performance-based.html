Performance-Based Routing

A DNS routing policy that directs users to the best-performing endpoint based on latency, network health, throughput, and real-time infrastructure metrics.

What is Performance-Based Routing?

Performance-based routing intelligently directs traffic to the infrastructure endpoint delivering the best user experience at a given moment. Unlike simple latency routing, it can evaluate multiple performance indicators such as response time, server load, packet loss, congestion, and availability.

⚡

How It Works

DNS systems continuously monitor application and network performance across multiple regions. Incoming user requests are dynamically routed to the endpoint currently providing the best overall performance.

📊

Metrics Evaluated

  • ✔ Network latency
  • ✔ Server response time
  • ✔ Packet loss
  • ✔ Infrastructure load
  • ✔ Regional congestion

Key Benefits

Faster User Experience

Users are automatically routed to the fastest and healthiest infrastructure available.

Intelligent Traffic Steering

Routing decisions adapt dynamically based on changing network and application conditions.

Better Reliability

Traffic automatically avoids overloaded or degraded infrastructure regions.
