Policy-Based Routing

Intelligent DNS routing using custom business and network rules.

What is Policy-Based Routing?

Policy-Based Routing routes traffic according to predefined rules instead of using only basic DNS responses.

Routing decisions can depend on factors such as source IP, ASN (Autonomous System Number), device type, user location, or custom application logic.

Routing Criteria

  • Source IP Address
  • ASN / ISP Network
  • Device Type
  • User Location
  • Application Rules

Common Benefits

  • Advanced traffic control
  • Improved performance
  • Enhanced security
  • Custom user experience
  • Enterprise-grade routing

How It Works

1

User Sends DNS Request

The DNS provider receives a request from a client device.

2

Policies Are Evaluated

The system checks routing rules such as IP, ASN, or device type.

3

Best Endpoint Selected

Traffic is routed to the endpoint matching the defined policy.

Example Policy

IF Device = Mobile
→ Route to Mobile Optimized Server

IF Source ASN = ISP-A
→ Route to Regional CDN

IF Source IP = Corporate Network
→ Route to Private Application Gateway
