Health-Check-Based Routing

Intelligent DNS routing that sends users only to healthy servers.

What is Health-Check-Based Routing?

Health-Check-Based Routing uses automated endpoint monitoring to determine whether a server or application instance is healthy.

DNS traffic is dynamically routed only to healthy resources, helping maintain high availability and better user experience.

Key Benefits

  • Improves uptime
  • Automatic failure detection
  • Better reliability
  • Supports disaster recovery

Common Use Cases

  • Failover routing
  • Weighted traffic routing
  • Multi-region applications
  • Cloud load balancing

How It Works

1

Health Check Runs

The DNS provider continuously checks server health.

2

Healthy Endpoints Selected

Only healthy instances remain available for routing.

3

Traffic Redirected

Traffic automatically shifts away from failed servers.

Example Scenario

Primary Server → Healthy ✅
Secondary Server → Standby ⏳

If Primary Fails ❌
DNS Automatically Routes Traffic → Secondary Server ✅
