Multi-Value Answer Routing

A DNS routing policy that returns multiple healthy IP addresses in response to a DNS query for improved availability, redundancy, and basic load distribution.

What is Multi-Value Answer Routing?

Multi-value answer routing allows a DNS server to return multiple IP addresses for a single domain name. Only healthy endpoints are included in the response, helping improve application availability and fault tolerance.

🌐

How It Works

Instead of returning only one IP address, the DNS service responds with multiple healthy endpoints. Client systems typically choose one of the returned addresses to establish a connection.

⚡

Key Advantages

  • ✔ High availability
  • ✔ Basic traffic balancing
  • ✔ Health-check integration
  • ✔ Improved fault tolerance
  • ✔ Simple global distribution

Example DNS Response 

Query:
example.com

DNS Response:
192.0.2.10
192.0.2.11
192.0.2.12

Benefits

Redundancy

Multiple servers ensure service continuity even if one endpoint becomes unavailable.

Simplicity

Easier to configure compared to advanced load balancing or geolocation routing systems.

Health Awareness

Unhealthy servers are automatically excluded from DNS responses using health checks.

Common Use Cases

  • 🌍 Global web applications
  • 🛡 Disaster recovery
  • ☁ Multi-cloud deployments
  • ⚡ Basic DNS load balancing
  • 📡 Highly available APIs
  • 🏢 Distributed enterprise systems
