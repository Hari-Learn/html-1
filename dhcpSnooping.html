DHCP Snooping

Network Security Feature for Layer 2 Protection

What is DHCP Snooping?

DHCP Snooping is a Layer 2 security feature that prevents unauthorized DHCP servers from distributing incorrect IP addresses to clients. It monitors DHCP traffic and filters out untrusted messages.

Why is it Important?

Without DHCP Snooping, rogue DHCP servers can assign malicious IP configurations, leading to Man-in-the-Middle attacks, traffic interception, or denial of service.

How It Works

  • Switch ports are classified as trusted or untrusted.
  • Trusted ports allow DHCP server responses.
  • Untrusted ports block DHCP offer/ack packets.
  • A DHCP binding table is maintained.

Key Features

  • Prevents rogue DHCP servers
  • Builds DHCP binding database
  • Supports Dynamic ARP Inspection
  • Enhances Layer 2 network security

Trusted vs Untrusted Ports

Trusted ports typically connect to legitimate DHCP servers. All other ports are untrusted by default and cannot send DHCP server messages.

Benefits

  • Improves network reliability
  • Prevents IP spoofing
  • Reduces security risks
  • Ensures correct IP allocation
