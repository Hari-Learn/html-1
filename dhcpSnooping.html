DHCP Snooping

CCNA Security Fundamentals

What is DHCP Snooping?

DHCP Snooping is a Layer 2 security feature implemented on switches to protect networks from rogue (unauthorized) DHCP servers. It allows the switch to monitor DHCP messages and permit only trusted devices to respond with IP address assignments.

Why is DHCP Snooping Needed?

  • Prevents attackers from assigning fake IP addresses
  • Stops Man-in-the-Middle (MITM) attacks
  • Protects default gateway and DNS information
  • Ensures only legitimate DHCP servers operate

How DHCP Snooping Works

Trusted Ports

Ports connected to legitimate DHCP servers are marked as trusted. These ports are allowed to send DHCP Offer and Acknowledgment messages.

Untrusted Ports

All other ports are untrusted by default. DHCP Offer and ACK messages from these ports are blocked. This prevents rogue DHCP servers.

DHCP Snooping Binding Table

DHCP Snooping builds a database called the binding table. This table records:

  • MAC address
  • Assigned IP address
  • VLAN ID
  • Port number

This table is used by other security features like Dynamic ARP Inspection (DAI) and IP Source Guard.

Attacks Prevented by DHCP Snooping

Rogue DHCP Server

Prevents attackers from distributing fake IP configurations.

Man-in-the-Middle

Stops attackers from redirecting traffic through themselves.

IP Spoofing

Prevents devices from using unauthorized IP addresses.

Why DHCP Snooping is Important for CCNA

DHCP Snooping is part of CCNA security fundamentals because it demonstrates how switches enforce security at Layer 2.

  • Understanding trusted vs untrusted ports
  • Identifying DHCP-based attacks
  • Knowing how switches protect IP addressing
  • Foundation for DAI and IP Source Guard topics
