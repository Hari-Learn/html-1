DHCP Snooping is a network security feature that protects your network from unauthorized (rogue) DHCP servers. It works at Layer 2 (Switch level) and monitors DHCP traffic.

‚ùì Why Do We Need It?

  • Prevents rogue DHCP servers
  • Stops wrong IP address assignment
  • Prevents Man-in-the-Middle attacks
  • Improves network security

‚öôÔ∏è How It Works

  • Switch ports are marked as Trusted or Untrusted
  • Trusted ports allow DHCP server responses
  • Untrusted ports block fake DHCP offers
  • Switch builds a DHCP Binding Table

üîê Trusted vs Untrusted Ports

Trusted Port: Connected to real DHCP Server.

Untrusted Port: Connected to users. Blocks DHCP server messages.

üìã DHCP Binding Table

The switch stores:

  • IP Address
  • MAC Address
  • VLAN
  • Interface

üíª Basic Configuration Example (Cisco)

Switch(config)# ip dhcp snooping
Switch(config)# ip dhcp snooping vlan 10
Switch(config)# interface g0/1
Switch(config-if)# ip dhcp snooping trust

‚úÖ Summary

DHCP Snooping protects your network by allowing DHCP replies only from trusted ports and blocking unauthorized DHCP servers. It is an important Layer 2 security feature in modern networks.
