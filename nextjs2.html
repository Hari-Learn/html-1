Next.js App Router Overview

The App Router (introduced in Next.js 13 and now standard in 2025) provides a powerful file-system-based routing system using the /app directory. It enables layouts, nested routing, React Server Components, streaming, and built-in data fetching.

1. App Router Project Structure

app/
 ├─ layout.js      (Root Layout)
 ├─ page.js        (Home Page)
 ├─ about/
 │   └─ page.js    (About Page)
 └─ api/
     └─ hello/route.js  (API Endpoint)

2. Routing Basics

A folder becomes a route. A page.js file returns UI.

// app/page.js
export default function Home() {
  return <h1>Welcome to Next.js App Router</h1>;
}

3. Layouts (Shared UI)

Layouts wrap all child routes and persist across navigation.

// app/layout.js
export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <header>My Header</header>
        {children}
      </body>
    </html>
  );
}

4. Server Components (Default)

Most components are Server Components by default, meaning they run on the server. They reduce bundle size and load faster. 

export default async function Page() {
  const data = await fetch("https://api.example.com/posts").then(r => r.json());
  return <pre>{JSON.stringify(data, null, 2)}</pre>;
}

5. Client Components

Add "use client" to use hooks, events, or interactive UI. 

"use client";
import { useState } from "react";

export default function Counter() {
  const [n, setN] = useState(0);
  return <button onClick={() => setN(n + 1)}>Count {n}</button>;
}

6. Data Fetching (Advanced Features)

Supports caching, ISR, revalidation, and streaming.

export async function Page() {
  const data = await fetch("https://api.example.com/users", {
    next: { revalidate: 60 } // ISR every 60 sec
  }).then(r => r.json());

  return <div>{data.name}</div>;
}

7. Loading UI (Streaming)

// app/dashboard/loading.js
export default function Loading() {
  return <p>Loading content...</p>;
}

8. Error & Not Found Pages

// app/error.js
export default function Error() {
  return <h2>Something went wrong!</h2>;
}

// app/not-found.js
export default function NotFound() {
  return <h1>404 - Page Not Found</h1>;
}

9. Route Groups

Used to organize folders without affecting the route URL.

app/
 └─ (dashboard)
    ├─ layout.js
    └─ page.js

10. Dynamic Routes

// app/posts/[id]/page.js
export default function Post({ params }) {
  return <h1>Post ID: {params.id}</h1>;
}
