DHCP Packets

Explanation of 8 DHCP packets and their functionality

1. DHCP Discover

The client broadcasts a DHCP Discover message to locate available DHCP servers on the network. This is the first step when a device wants to obtain an IP address.

2. DHCP Offer

The DHCP server responds with a DHCP Offer message, offering an IP address and other network configuration details such as subnet mask and lease time.

3. DHCP Request

The client sends a DHCP Request message to request the offered IP address. This message is usually broadcast to inform all servers which offer was accepted.

4. DHCP Acknowledge (ACK)

The DHCP server sends a DHCP ACK message confirming the IP address lease. The client can now use the assigned IP configuration.

5. DHCP Negative Acknowledge (NAK)

The DHCP server sends a DHCP NAK message if the requested IP address is invalid or no longer available. The client must restart the DHCP process.

6. DHCP Release

The client sends a DHCP Release message to inform the server that it no longer needs the assigned IP address, allowing it to be reused.

7. DHCP Decline

The client sends a DHCP Decline message when it detects that the offered IP address is already in use on the network.

8. DHCP Inform

The DHCP Inform message is sent by a client that already has an IP address but wants additional configuration information such as DNS server details.
