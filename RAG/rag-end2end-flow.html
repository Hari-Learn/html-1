WHAT IS RAG?

RAG is Like an Open-Book Exam

Imagine a student taking a test.

Instead of remembering everything, the student can first search books, then write the answer.

RAG works the same way.

  • 🔍 First it searches for information
  • 🧠 Then AI reads the best parts
  • ✨ Finally it generates the answer

End-to-End RAG Pipeline

1

Documents Are Added

First, we give RAG lots of information:

📄 PDFs
🌐 Websites
📚 Notes & Docs
2

Split Into Small Chunks

Big documents are broken into small pieces called chunks so AI can search them easily.

3

Convert Text into Embeddings

AI converts text into special number patterns called embeddings.

🧠 Embeddings help computers understand the meaning of text.

4

Store in Vector Database

The embeddings are saved inside a vector database so AI can search them quickly later.

5

User Asks a Question

Example:

“Why do airplanes fly?”

6

Similarity Search Happens

AI searches the vector database to find chunks with similar meaning.

7

ReRanking + MMR Improve Results

RAG now improves the search results:

🔄 ReRanking

Moves best documents to the top.

🌈 MMR

Removes repetitive results and keeps variety.

8

AI Reads the Best Context

The best retrieved chunks are sent to the Large Language Model.

9

Final Answer Generated

The AI now creates a smarter answer using both:

🧠 AI Knowledge
📚 Retrieved Context
