CCNA Security Fundamentals

CIA Triad, Threats, Attacks, and Mitigations

Why This Matters for CCNA

Security is an essential part of networking. CCNA candidates must understand key security concepts like the CIA Triad, threats, vulnerabilities, exploits, and common attacks. These fundamentals help network engineers design secure networks and protect against potential attacks.

Confidentiality

Protecting data from unauthorized access.

Example: Using encryption for data in transit.

Integrity

Ensuring data is accurate and unaltered.

Example: Hashing files to detect tampering.

Availability

Ensuring systems and data are accessible when needed.

Example: Redundant servers and DDoS protection.

Threats, Vulnerabilities, Exploits & Mitigation

Threats

Potential dangers that can harm a network or system.

Example: Malware, hacking attempts, phishing.

Vulnerabilities

Weaknesses in a system that threats can exploit.

Example: Unpatched software, weak passwords.

Exploits

Techniques attackers use to take advantage of vulnerabilities.

Example: SQL injection, buffer overflow attacks.

Mitigation

Actions to reduce risk or prevent attacks.

Example: Firewalls, strong passwords, updates, security policies.

Common Attacks

DoS / DDoS

Overloading systems to make them unavailable.

Mitigation: Rate limiting, firewalls, redundancy.

Spoofing

Pretending to be someone else to gain access.

Mitigation: Authentication, secure protocols.

Malware

Malicious software like viruses, worms, ransomware.

Mitigation: Antivirus, updates, user training.

Social Engineering

Tricking users to reveal sensitive info.

Mitigation: Awareness training, verification processes.
