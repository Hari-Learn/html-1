What is an ACL? (CCNA)

An Access Control List (ACL) is a set of rules used to permit or deny traffic based on conditions.

In CCNA, ACLs answer this question: "Should this traffic be allowed or blocked?"

Basic ACL Logic

  • ACLs are checked top to bottom
  • First matching rule is applied
  • If no rule matches â†’ implicit deny

What Does an ACL Check?

Source IP

Who is sending traffic

Destination IP

Where traffic is going

Protocol

TCP, UDP, ICMP

Port Number

HTTP (80), HTTPS (443), FTP (21)

How Palo Alto Firewall Uses ACL Concepts

Palo Alto does not use the word ACL. Instead, it uses Security Policies.

CCNA ACL Palo Alto Firewall
Permit / Deny Allow / Deny
Source IP Source Address
Destination IP Destination Address
Protocol & Port Application / Service

CCNA ACL Example

Requirement:

Allow PC1 (192.168.1.10) to access Web Server (192.168.1.100) using HTTP.

access-list 100 permit tcp host 192.168.1.10 host 192.168.1.100 eq 80

In Palo Alto, the same rule is created as a Security Policy.

Important CCNA Exam Points

  • ACLs work on permit / deny logic
  • Rules are read top to bottom
  • Implicit deny always exists
  • Palo Alto policies follow same logic but are more advanced
