OSPF (Open Shortest Path First) – Routing Protocol Guide

What is OSPF?

OSPF (Open Shortest Path First) is a link-state routing protocol used in IP networks. It is an Interior Gateway Protocol (IGP) and is commonly used in enterprise and service provider networks.

What is a Distance Vector Routing Protocol?

Distance Vector routing protocols share routing information with neighbors based on distance (metric) and direction (next hop).

  • Examples: RIP, IGRP
  • Uses periodic updates
  • Slower convergence

What is a Link-State Routing Protocol?

Link-state routing protocols maintain a complete map of the network topology. Routers exchange link-state information and calculate the shortest path using the SPF (Dijkstra) algorithm.

  • Examples: OSPF, IS-IS
  • Fast convergence
  • Uses LSDB

What is Router ID?

Router ID is a 32-bit unique identifier used to identify a router in an OSPF network.

Router ID selection order:

  1. Manually configured Router ID
  2. Highest IP address on Loopback interface
  3. Highest IP address on physical interface

OSPF Identification

OSPF uses Router ID, Area ID, and Process ID to identify routers and routing domains within the OSPF network.

What is LSDB?

LSDB (Link State Database) contains complete topology information of the OSPF area. All routers in the same area have an identical LSDB.

What is Neighbor Table?

Neighbor Table stores information about directly connected OSPF neighbors, including their Router ID, state, and timers.

What is an OSPF Area?

An OSPF area is a logical grouping of routers. Areas help reduce routing table size and improve performance.

  • Area 0 is Backbone Area
  • All areas must connect to Area 0

How to Configure OSPF (Basic to Intermediate)

Basic Configuration:

router ospf 1
network 192.168.1.0 0.0.0.255 area 0

Intermediate Configuration:

router ospf 1
router-id 1.1.1.1
network 10.0.0.0 0.0.0.255 area 0
network 20.0.0.0 0.0.0.255 area 1

What Happens When OSPF is Enabled?

  • Hello packets are generated
  • Neighbor discovery starts
  • LSDB synchronization occurs
  • SPF algorithm calculates best path

Why OSPF Uses Multicast IP 224.0.0.5?

OSPF uses Class D multicast address 224.0.0.5 to send Hello and routing updates to all OSPF routers on the network.

OSPF Neighbor Relationship Conditions

  • Same Area ID
  • Same Hello and Dead timers
  • Same network type
  • Unique Router ID

Hello Time and Dead Time

Hello Time: Interval at which Hello packets are sent (default 10 seconds).

Dead Time: Time to wait before declaring a neighbor down (default 40 seconds).
