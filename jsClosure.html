Closures & Important Questions You Must Know

What is a Closure?

A closure is a function that remembers variables from its outer scope even after the outer function has finished executing. 

function outer() {
  let count = 0;

  return function inner() {
    count++;
    console.log(count);
  }
}

const counter = outer();
counter(); // 1
counter(); // 2

Why Closures are Important?

  • Data encapsulation
  • Function factories
  • Maintaining state in async operations
  • Used in callbacks and event handlers

Top JavaScript Interview Questions

1. What is hoisting?

Hoisting is JavaScript's default behavior of moving declarations to the top of the scope.

2. Difference between var, let, const?

var is function-scoped, let and const are block-scoped. const cannot be reassigned.

3. What is event delegation?

A technique where a parent element handles events for its children using event bubbling.

4. What is the difference between == and ===?

== compares values with type coercion, === compares both value and type.

5. What is a promise?

A promise represents a value that may be available now, later, or never.
