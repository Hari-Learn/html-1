How to Isolate One System in Layer 2 Switch (Without VLAN)

1. Port Isolation / Protected Port

Port Isolation allows devices connected to specific switch ports to be blocked from communicating with each other while still allowing access to the uplink or router port.

Cisco Example:

Switch(config)# interface fa0/2
Switch(config-if)# switchport protected

2. MAC Access Control List (MAC ACL)

If supported, you can block communication using MAC-based Access Control Lists. This allows denying traffic from a specific device’s MAC address.

Example:

Switch(config)# mac access-list extended BLOCK_PC1
Switch(config-ext-macl)# deny any host 0001.aaaa.bbbb
Switch(config-ext-macl)# permit any any

3. Port Security (Device Restriction)

Port Security limits which devices can connect to a switch port. It prevents unauthorized devices but does not fully isolate ports from each other.

Example:

Switch(config)# interface fa0/2
Switch(config-if)# switchport port-security
Switch(config-if)# switchport port-security maximum 1

✔ Best Recommended Method Without VLAN: Port Isolation / Protected Port
