Basics You Must Learn in Next.js

Next.js is a React framework that makes building production-ready applications easier. Here are the essential things every beginner should learn.

1. Install Next.js

Install Node.js and create a new Next.js app:

npx create-next-app@latest my-next-app
cd my-next-app
npm run dev

2. Understand the App Router

The App Router (using the /app folder) is the latest and recommended way to build Next.js apps. It supports layouts, nested routes, and server components.

3. Learn Pages & Routing

Every folder inside /app becomes a route. Example: app/about/page.js → /about

4. Layouts

Layouts allow shared UI such as headers and footers. Example: 

export default function Layout({ children }) {
  return (
    <html>
      <body>
        <header>My Site</header>
        {children}
      </body>
    </html>
  );
}

5. Server vs Client Components

By default, components are **Server Components** (faster, no JS on client). Use "use client" at the top if you need client-side JS, such as for interactive UI.

6. Data Fetching

You can fetch data directly in server components using fetch(). 

export default async function Page() {
  const data = await fetch("https://api.example.com/posts").then(r => r.json());
  return <div>{JSON.stringify(data)}</div>;
}

7. API Routes (Route Handlers)

Create backend API inside the app folder: 

// app/api/hello/route.js
export function GET() {
  return Response.json({ message: "Hello API" });
}

8. Rendering Types

  • SSR – Server-Side Rendering
  • SSG – Static Site Generation
  • ISR – Incremental Static Regeneration
  • CSR – Client-Side Rendering

9. Deployment

Deploy easily on:

  • Vercel (best)
  • Netlify
  • AWS
  • Docker

Simple Tutorial: Your First Page

Create a file:

app/hello/page.js

Add this:

export default function Hello() {
  return <h1>Hello from Next.js!</h1>;
}

Now visit http://localhost:3000/hello — that’s your first custom route!

You’re ready to build!

With these basics, you can now create full apps, blogs, dashboards, API backends, and more using Next.js.
