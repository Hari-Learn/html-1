OSPF Routing Protocol

Complete Concepts Explained Using Colorful Cards

What is OSPF?
OSPF (Open Shortest Path First) is a link-state interior gateway protocol used to find the shortest path in large enterprise networks using the SPF algorithm.
Distance Vector Routing Protocol
Distance Vector protocols share routes based on distance and direction.
  • Example: RIP
  • Uses periodic updates
  • Slow convergence
Link-State Routing Protocol
Link-state protocols build a full network topology using LSAs and calculate shortest paths using Dijkstra’s SPF algorithm.
Router ID
Router ID is a 32-bit unique identifier for OSPF routers.
  1. Manual configuration
  2. Highest loopback IP
  3. Highest physical IP
OSPF Identification
OSPF identifies routers using:
  • Router ID
  • Area ID
  • Process ID
LSDB (Link State Database)
LSDB stores complete topology information of an OSPF area. All routers in the same area have an identical LSDB.
Neighbor Table
Stores information about directly connected OSPF neighbors including Router ID, state, and timers.
OSPF Area
OSPF areas divide the network to improve scalability.
  • Area 0 = Backbone
  • All areas connect to Area 0
OSPF Configuration 
router ospf 1
network 192.168.1.0 0.0.0.255 area 0
router-id 1.1.1.1
What Happens When OSPF Starts?
  • Hello packets sent
  • Neighbors discovered
  • LSDB synchronized
  • SPF calculation
Multicast IP 224.0.0.5
OSPF uses Class D multicast address 224.0.0.5 to communicate with all OSPF routers on a network.
Neighbor Adjacency Conditions
  • Same Area ID
  • Same Hello & Dead time
  • Same network type
  • Unique Router ID
