Subnetting Cheat Sheet

Must-memorize subnetting tables & concepts for CCNA exam

Why Subnetting is Critical for CCNA

  • Almost every CCNA exam includes subnetting questions
  • Required for routing, VLANs, ACLs, and IP planning
  • Tests speed, accuracy, and concept clarity

Magic Number (Fast Subnetting Trick)

Magic Number = 256 − subnet mask value (in the interesting octet)

Example

Mask: 255.255.255.192

Magic Number: 256 − 192 = 64

Subnets

0, 64, 128, 192

Must-Know Subnetting Concepts

  • Network Address = first IP in subnet
  • Broadcast Address = last IP in subnet
  • Usable hosts = Total − 2
  • /30 used for point-to-point links
  • Private IP ranges must be memorized

Private IP Address Ranges (EXAM FAVORITE)

  • Class A: 10.0.0.0 – 10.255.255.255
  • Class B: 172.16.0.0 – 172.31.255.255
  • Class C: 192.168.0.0 – 192.168.255.255

CCNA Subnetting Exam Tips

  • Memorize /24 to /30 table perfectly
  • Practice speed – CCNA is time-based
  • Always find the interesting octet first
  • Draw subnet table during exam
