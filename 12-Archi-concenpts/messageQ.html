Message Queue

A system design pattern that enables asynchronous communication between services using a queue-based messaging system.

Message Flow

Producer

Sends Messages

➡️
Queue

Stores Messages

➡️
Consumer

Processes Messages

How It Works

  • Producer sends a message to the queue
  • Messages are stored temporarily
  • Consumer retrieves messages when ready
  • Processing happens asynchronously

Benefits

  • Decouples services
  • Improves reliability
  • Handles traffic spikes
  • Scales easily

Real-World Examples

  • Email sending systems
  • Order processing in e-commerce
  • Background job processing
  • Notification systems

Popular Tools

  • RabbitMQ
  • Apache Kafka
  • Amazon SQS
  • Redis Queue
