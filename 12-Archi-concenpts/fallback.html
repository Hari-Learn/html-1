Fallback Pattern

What is it?

The Fallback Pattern is a resilience design pattern used in microservices to provide an alternative response when a service fails. Instead of breaking the application, it gracefully returns backup data, default responses, or cached values.

ELI10 Explanation

It’s like carrying an umbrella ☔. If it rains (failure happens), you don’t panic — you already have a backup plan ready!

Why is it Important?

  • Prevents poor user experience
  • Keeps apps partially functional
  • Improves reliability and availability
  • Handles temporary failures smoothly

Types of Fallbacks

  • Static: Fixed message
  • Cached: Last known data
  • Default: Safe values
  • Alternative Service: Backup API

Real-World Examples

  • Weather app shows cached data 🌦️
  • E-commerce shows popular products 🛒
  • Video apps show thumbnails 🎥

Simple Flow

Request → Service Call → ❌ Failure → Fallback → ✅ Response

Code Example 

function getData() {
  try {
    return callService();
  } catch (e) {
    return "Fallback response";
  }
}

When to Use

  • External API calls
  • Distributed systems
  • Non-critical features

When NOT to Use

  • Critical operations (payments 💳)
  • When incorrect data is risky
