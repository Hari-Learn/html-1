Fallback Pattern

What is it?

A resilience pattern that provides a backup response when a service fails, ensuring the system continues to function gracefully.

ELI10

Like carrying an umbrella ☔ — if something goes wrong, you already have a backup.

Why Important?

  • Better user experience
  • High availability
  • Graceful failure handling

Types

  • Static response
  • Cached data
  • Default values
  • Backup service

Examples

  • Weather app shows old data 🌦️
  • Shopping shows popular items 🛒
  • Video app shows thumbnails 🎥

Flow

Request → Service → ❌ Fail → Fallback → ✅ Response

Code Example 

function getData() {
  try {
    return callService();
  } catch (e) {
    return "Fallback response";
  }
}

Use When

  • External APIs
  • Distributed systems
  • Non-critical features

Avoid When

  • Payments 💳
  • Critical data flows
