Database Sharding

Scaling databases horizontally

What is Sharding?

Sharding is a database architecture pattern where data is split across multiple servers called shards. Each shard holds a portion of the total dataset.

Why Use Sharding?

  • Improves database performance
  • Supports large-scale applications
  • Distributes data across servers

Sharding Strategies

  • Range-Based Sharding
  • Hash-Based Sharding
  • Directory-Based Sharding
