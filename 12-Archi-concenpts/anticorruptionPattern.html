Anti-Corruption Layer (ACL)

A design pattern that protects your new system from messy, outdated, or incompatible systems.

Explain Like I'm 10 👶

Imagine two people speaking different languages. They can’t understand each other directly.

So they use a translator 🗣️. The translator listens, converts the message, and passes it along.

👉 Anti-Corruption Layer = that translator It keeps your new system clean and safe from confusing old systems.

What is it?

An Anti-Corruption Layer (ACL) is a middle layer that sits between a new system and an old (legacy) system.

It translates data, formats, and logic so the new system doesn’t get “polluted” by bad design or outdated structures.

How It Works

  1. New system sends a clean request
  2. ACL converts it into legacy format
  3. Old system processes it
  4. ACL converts response back into clean format
  5. New system receives simple, consistent data

Why Do We Need It?

  • Legacy systems are messy or inconsistent
  • Different data formats (XML vs JSON)
  • Old naming conventions
  • Hard to change legacy code

Benefits

  • 🧼 Keeps new system clean
  • 🔄 Easy to replace legacy later
  • 🧠 Simplifies logic in new system
  • ⚠️ Reduces bugs from bad data

Real-World Example

A modern e-commerce app wants to use a 20-year-old banking system.

Instead of directly connecting:

  • The app talks to the ACL
  • The ACL converts requests into old banking format
  • The banking system responds
  • The ACL cleans and sends it back

Another Simple Analogy

Think of a phone charger adapter 🔌:

Your new phone uses USB-C, but the wall socket is different. The adapter makes them work together safely.

👉 ACL = adapter between incompatible systems
