Circuit Breaker Pattern

Improving reliability in distributed systems

What is Circuit Breaker?

The Circuit Breaker pattern prevents an application from repeatedly trying to execute an operation that is likely to fail. It works like an electrical circuit breaker that stops the flow of current when a fault occurs.

How It Works

  • Closed State – Requests flow normally
  • Open State – Requests are blocked after repeated failures
  • Half-Open State – Allows limited requests to test recovery

Benefits

  • Prevents cascading failures
  • Improves system stability
  • Allows failing services time to recover
