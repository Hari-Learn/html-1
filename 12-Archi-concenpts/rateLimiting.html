Rate Limiting

Explained simply (ELI10)

❓ Why do we need Rate Limiting?

Imagine a popular ice cream shop 🍦 with only one worker.

If too many people order at once:

  • The worker gets overwhelmed 😵
  • Orders become slow 🐢
  • Some customers may never get served 😞

So the shop allows only a limited number of orders per minute. That’s exactly what rate limiting does for servers.

📦 What is Rate Limiting?

Rate limiting controls how many requests a user or system can make within a certain time.

User
→
Limit Check
→
Server

If the limit is exceeded → ❌ Request is blocked

⚙️ How does it work?

🪣 Token Bucket

Tokens fill over time. Each request uses one.

💧 Leaky Bucket

Requests are processed at a steady speed.

⏱ Fixed Window

Limits requests per time window (e.g., 100/min).

🔄 Simple Flow

Client
→
Rate Limiter
→
Server

The rate limiter decides whether to allow or block requests.

⚡ In One Line

Rate limiting = controlling how fast users can send requests to keep systems safe and stable.
