Publish–Subscribe

Publish–Subscribe (Pub/Sub) is a messaging pattern where publishers send messages without knowing the receivers, and subscribers receive messages based on topics.

How It Works

  • Publisher sends a message to a topic
  • Message broker distributes it
  • Subscribers listening to the topic receive the message
  • Multiple subscribers can receive the same message

Advantages

  • Loose coupling between systems
  • Scalable event-driven architecture
  • Real-time data updates
  • Supports multiple consumers
