Event Sourcing Pattern

📌 What is it?

Event Sourcing stores the system state as a sequence of events instead of just saving the latest state.

🚀 Why use it?

  • Full history of changes
  • Easy debugging & auditing
  • Rebuild state anytime

⚠ Challenges

  • More complex design
  • Event versioning issues
  • Storage can grow large

🧒 Explain Like I'm 10

Imagine your bank account:

  • You don’t just store your current balance 💰
  • You keep a list of all transactions 🧾
  • Deposit → Withdraw → Transfer

If needed, you can replay all transactions to calculate your balance again. 👉 That’s exactly how Event Sourcing works!

How It Works

  1. User performs an action (e.g., place order)
  2. System creates an event (OrderCreated)
  3. Event is stored in an event store
  4. State is rebuilt by replaying all events
  5. Read models can be generated for fast queries
