Publish-Subscribe Architecture

Publish-Subscribe (Pub/Sub) is a messaging pattern where publishers send messages without knowing subscribers, and subscribers receive messages based on topics.

Key Concepts

  • Decoupled communication
  • Message broker handles delivery
  • Topics or channels for routing

Components

  • Publisher
  • Subscriber
  • Broker (e.g., Kafka, RabbitMQ)

Use Cases

  • Event-driven systems
  • Real-time notifications
  • Microservices communication
