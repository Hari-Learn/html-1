Immutable Deployment

Imagine you built a toy house, but instead of fixing it, you always build a brand new one.

That’s how immutable deployment works—you never change old servers.

How it works:

  • Create a brand new version of the app
  • Start new servers with it
  • Delete the old servers

This avoids bugs caused by leftover changes or broken updates.

Simple example:

Like replacing your old phone with a new one instead of repairing it again and again.
