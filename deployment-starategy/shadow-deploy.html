Shadow Deployment

Imagine you built a new version of your app, but you're not sure if it works perfectly yet.

So instead of showing it to users, you let it quietly run in the background like a "shadow".

How it works:

  • Users use the old app normally
  • The same requests are secretly sent to the new app
  • But users never see the new app

This helps developers check if the new version behaves correctly without any risk.

Simple example:

Like testing a new robot chef in the kitchen, but not serving its food yet—just comparing it with the real chef.
