OSPF Neighbor States

Understanding OSPF link and adjacency formation process

What is OSPF?

Open Shortest Path First (OSPF) is a link-state routing protocol used in IP networks. OSPF routers form neighbor relationships and progress through several states before reaching full adjacency and exchanging routing information.

1. Link Down / Link Up

Link Down: No OSPF Hello packets are received. The interface is down or not configured for OSPF.

Link Up: The interface becomes active, and the router starts sending OSPF Hello packets to discover neighbors.

2. Init State

The router receives a Hello packet from a neighbor, but the router’s own Router ID is not listed in the neighbor’s Hello packet.

This indicates one-way communication.

3. 2-Way State

The router sees its own Router ID in the neighbor’s Hello packet, confirming two-way communication.

On broadcast and NBMA networks, DR (Designated Router) and BDR elections occur at this stage.

4. ExStart State

Routers decide which one will be the master and which will be the slave for database synchronization.

The master controls the exchange process using sequence numbers.

5. Exchange State

Routers exchange Database Description (DBD) packets.

These packets describe the contents of the link-state database without sending the full LSAs.

6. Loading State

Routers request missing or outdated Link-State Advertisements (LSAs) using Link-State Request packets.

The requested LSAs are sent in Link-State Update packets.

7. Full State

The routers have fully synchronized link-state databases.

This is the final and stable state where routers can calculate routes using the SPF (Dijkstra) algorithm.
