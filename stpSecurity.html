Spanning Tree Protection Mechanisms

These features enhance Spanning Tree Protocol (STP) stability and protect your network from loops and unauthorized topology changes.

Loop Guard

Prevents alternate or root ports from transitioning to forwarding when BPDUs are no longer received due to unidirectional link failure.

  • Protects against unidirectional link issues
  • Works on non-designated ports
  • Places port in loop-inconsistent state
switch(config)# spanning-tree guard loop

Root Guard

Prevents an access switch from becoming the root bridge by blocking superior BPDUs.

  • Applied to designated ports
  • Stops unauthorized root bridge changes
  • Places port in root-inconsistent state
switch(config-if)# spanning-tree guard root

BPDU Protection

Secures PortFast-enabled access ports by shutting them down if BPDUs are received.

  • Used on edge/access ports
  • Disables port upon BPDU reception
  • Protects against rogue switches
switch(config-if)# spanning-tree bpduguard enable
