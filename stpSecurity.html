Spanning Tree Protection Mechanisms

Loop Guard, Root Guard, and BPDU Protection Explained

🔁 Loop Guard

Loop Guard prevents switching loops caused by unidirectional link failures. It ensures that a port does not incorrectly transition to the forwarding state when BPDUs are no longer received.

Key Points:

  • Works on non-designated ports.
  • Prevents alternate/backup ports from becoming forwarding unexpectedly.
  • Places the port into loop-inconsistent state if BPDUs stop.
switch(config)# spanning-tree guard loop

🌳 Root Guard

Root Guard prevents a port from becoming the root port. It protects the network from unauthorized switches attempting to become the root bridge.

Key Points:

  • Applied to designated ports.
  • Prevents superior BPDUs from changing the root bridge.
  • Puts the port into root-inconsistent state.
switch(config-if)# spanning-tree guard root

🛡 BPDU Protection

BPDU Protection secures edge ports by shutting them down if BPDUs are received. This prevents accidental or malicious switch connections on access ports.

Key Points:

  • Typically used with PortFast-enabled ports.
  • Automatically disables the port upon receiving BPDUs.
  • Helps protect against rogue switches.
switch(config-if)# spanning-tree bpduguard enable
