Route Poisoning & Split Horizon

Loop prevention mechanisms in Distance Vector routing protocols (RIP)

Distance Vector Routing (Quick Context)

Distance Vector protocols (like RIP) make routing decisions based on:

  • Distance – metric (hop count in RIP)
  • Vector – direction (next-hop router)

Routers periodically share their entire routing table with neighbors — which makes loops possible.

The Problem: Routing Loops

If a network goes down and routers don’t learn about it correctly, they may keep advertising bad routes to each other.

This causes the classic issue: Count-to-Infinity.

Split Horizon

Split Horizon is a simple rule:

❝ A router does NOT advertise a route back out the interface it was learned on ❞

This prevents a router from telling a neighbor: “Hey, I can reach this network through you.”

Route Poisoning

Route Poisoning is used when a route becomes unreachable.

Instead of silently removing the route, the router:

  • Advertises the route with an infinite metric
  • In RIP, infinity = 16 hops
Network: 10.1.1.0 → Metric 16 (Unreachable)

How Split Horizon & Route Poisoning Work Together

Split Horizon

Prevents bad information from being advertised in the first place.

Route Poisoning

Quickly tells all routers that a route is dead.

What You Must Understand as a Network Engineer

  • Distance Vector protocols are simple but slow to converge
  • Routing loops are a real risk without prevention mechanisms
  • Split Horizon is enabled by default on most interfaces
  • Route Poisoning speeds up failure awareness
  • Modern networks prefer link-state protocols, but concepts still matter

Split Horizon prevents lies. Route Poisoning kills bad routes fast. Understand both to master Distance Vector routing
