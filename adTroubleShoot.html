2. Common AD Symptoms Seen by Network Engineers

  • Users can connect to network but cannot log in
  • VPN connects but domain resources are inaccessible
  • Slow logins across WAN links
  • Group Policy not applying
  • Intermittent authentication failures

3. What AD Depends On (Network View)

DNS (MOST IMPORTANT)

  • Clients must use AD DNS servers
  • SRV records must be reachable
  • No public DNS for domain joins

IP Connectivity

  • Client ‚Üî DC reachability
  • Routing between sites
  • No asymmetric paths

Time Synchronization

  • Kerberos requires accurate time
  • Max 5-minute skew
  • NTP must be consistent

Firewall Rules

  • LDAP / Kerberos ports open
  • RPC dynamic ports allowed
  • No DC-to-DC blocking

4. AD Ports Every Network Engineer Must Know

Service Port Why it matters
DNS 53 (TCP/UDP) DC discovery
Kerberos 88 Authentication
LDAP 389 Directory queries
LDAPS 636 Secure LDAP
RPC 135 + Dynamic DC communication

5. AD Troubleshooting Flow (Network Engineer Style)

  1. Check IP & Routing
    Can client reach DC?
  2. Check DNS
    Is client using AD DNS? Can it resolve DC?
  3. Check Time Sync
    Is time within 5 minutes?
  4. Check Firewall
    Are required ports allowed?
  5. Check Site & Subnet
    Is client talking to nearest DC?

6. Common Mistakes New Network Engineers Make

  • Using Google DNS instead of AD DNS
  • Blocking RPC dynamic ports
  • Ignoring time synchronization
  • Assuming VPN = full AD connectivity
  • Forgetting site-to-site latency
