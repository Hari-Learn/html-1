What is DNAT?

DNAT means Destination Network Address Translation.

Think of an apartment building ğŸ¢ with one main gate. Visitors come to the gate, and the security guard sends them to the correct flat inside.

DNAT changes the destination IP of incoming traffic.

Why Do We Need DNAT?

  • To allow internet users to reach internal servers
  • To publish websites behind a firewall
  • To hide real internal IP addresses
  • To use one public IP for many services

Interview-Ready Answers ğŸ¯

  • Q: What is DNAT?
    A: DNAT translates the destination IP of incoming traffic to an internal private IP.
  • Q: Where is DNAT used?
    A: Publishing internal services like web or mail servers to the internet.
  • Q: Difference between SNAT and DNAT?
    A: SNAT changes source IP (outgoing), DNAT changes destination IP (incoming).
  • Q: Can SNAT and DNAT work together?
    A: Yes, very commonly in firewall configurations.
