Server virtualization allows multiple virtual machines (VMs) to run on a single physical server using a hypervisor. Each VM operates independently with its own OS and resources.

  • Efficient hardware utilization
  • Isolation between virtual machines
  • Scalability and resource flexibility
  • Common hypervisors: VMware ESXi, Hyper-V, KVM

Containers package applications and their dependencies together, sharing the host OS kernel. Unlike VMs, containers are lightweight and start quickly.

  • Lightweight and fast deployment
  • Efficient resource consumption
  • Popular tools: Docker, Kubernetes
  • Ideal for microservices architecture

VRF allows multiple separate routing tables to exist on the same router. This enables network segmentation without requiring multiple physical routers.

  • Logical separation of networks
  • Improved security and traffic isolation
  • Commonly used in MPLS environments
  • Useful for multi-tenant architectures
