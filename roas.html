VLANs are created to separate networks and reduce broadcasts. But once VLANs are created, devices in different VLANs cannot communicate at all.

üß† Core Networking Rule (Very Important)

Devices in different IP networks need a ROUTER to communicate.

VLANs create different IP networks. Therefore, a router is mandatory.

‚ùì Why Not Use Multiple Router Interfaces?

Traditional routing would require:

  • One router interface per VLAN
  • More cables
  • Higher cost
  • Poor scalability

This becomes impractical as VLAN count increases.

‚úÖ Solution: Router-on-a-Stick (ROAS)

Router-on-a-Stick allows a single router interface to route traffic between multiple VLANs using subinterfaces.

How ROAS Works:

  • One physical router interface
  • Multiple logical subinterfaces
  • Each subinterface = one VLAN
  • 802.1Q trunk carries all VLAN traffic
