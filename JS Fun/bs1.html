Burp Suite — Cheatsheet

Fullscreen, no-scroll layout — compact & readable. Test only with permission.

Overview

What it is

Burp Suite (PortSwigger) — intercepting proxy + toolset for testing web apps: Proxy, Target, Repeater, Intruder, Scanner (Pro), Decoder, Sequencer, Extender.

Core Tools

  • Proxy — intercept/modify HTTP(S)
  • Target — site map & inspection
  • Repeater — manual tampering & resend
  • Intruder — automated payloads/fuzzing
  • Scanner (Pro) — active & passive scans

Typical Quick Workflow

  1. Point browser to proxy 127.0.0.1:8080 and install Burp CA cert for HTTPS.
  2. Browse app to populate Target; use Spider/content discovery for mapping.
  3. Send interesting requests to Repeater/Intruder/Scanner as needed.
  4. Analyze with Decoder, Sequencer and Comparer; export reports if required.

Shortcuts & Quick Actions

Keys & tips

  • Ctrl+R → Send to Repeater
  • Ctrl+I → Send to Intruder
  • Right-click a request → Send to Repeater/Intruder/Scanner/Comparer
  • Proxy → HTTP history → search / export selected

Intruder Modes

  • Sniper — single param focused
  • Clusterbomb — combinatorial multi-position (noisy)
  • Pitchfork — parallel aligned payload sets
  • Turbo — fast (use cautiously)

Useful BApp Extensions

Install via Extender → BApp Store

  • Logger++ — enhanced logging
  • JSON / Pretty Print — response formatting
  • Autorize — authorization checks
  • Turbo Intruder (advanced) — high-performance fuzzing

Headers & Checks

  • Host, Cookie, Referer, User-Agent — often relevant
  • Compare status codes, body length, timing, header presence
  • Use Comparer to diff responses quickly

Safety & Legal

Be responsible

Only test systems you own or have explicit permission to test. Unauthorized scanning/exploitation is illegal. Keep logs; avoid destructive actions.

Quick Reference

Proxy

Intercept toggle → capture & edit requests. Use "Match & Replace" for automated edits.

Repeater

Manual tampering: change params/headers/body then resend to observe behavior.

Intruder

Set positions → payload sets → attack type → run (careful: noisy).

Scanner (Pro)

Passive & active vulnerability checks — only on authorized targets.
