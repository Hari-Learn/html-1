Understanding OpenVAS / GVM

What is OpenVAS?

OpenVAS (Open Vulnerability Assessment System) is a free, open-source vulnerability scanner. It's now part of the Greenbone Vulnerability Management (GVM) system used to assess and report on network security issues.

GVM Components

  • OpenVAS Scanner: Executes tests against target systems.
  • GVMD: Manages scan configs, targets, tasks, and results.
  • GSA: Web interface for users (GUI).
  • Feeds: Updated vulnerability test data (NVTs).

How It Works

The scanner sends probes to discover systems, services, and known vulnerabilities. Results are analyzed and matched with known CVEs and security advisories.

Scans can be authenticated (with credentials) or unauthenticated (external probes).

Common Output

Each scan generates a report showing vulnerabilities, severity levels (Low/Medium/High/Critical), CVE IDs, affected services, and remediation advice.

Example Severities

  • Low: Minor issues, minimal risk.
  • Medium: Could be exploited with some effort.
  • High: Likely to be exploited, serious impact.
  • Critical: Immediate attention needed!

Use Cases

  • Internal security audits
  • Network vulnerability assessments
  • Compliance verification (e.g., ISO, NIST)

Best Practices

  • Scan only systems you own or have permission to assess.
  • Keep your feed updated (daily/weekly).
  • Prioritize remediation based on severity + impact.

Ethical & Legal Note

Running OpenVAS against systems without permission is illegal and unethical. Always get written authorization before scanning.
