What is CSRF?

CSRF stands for Cross-Site Request Forgery. It's a type of attack that tricks a user into submitting a request they did not intend. It typically targets state-changing requests (like updating a password or making a purchase) without the user’s knowledge.

Why is it Dangerous?

CSRF exploits the trust a site has in a user's browser. If a user is logged into a site, a malicious site can send a request on their behalf — using cookies automatically sent by the browser — without their consent.

How Does it Work?

Imagine you're logged into example.com, and visit a malicious site with this code: 

<img src="https://example.com/delete-account" />

Your browser sends the request including your session cookies — and your account gets deleted.

How to Prevent CSRF

  • Use CSRF Tokens: Generate a unique token per form/session and validate it on the server.
  • SameSite Cookies: Set SameSite=Strict or Lax to block cross-origin requests.
  • Double Submit Cookies: Send a CSRF token in a cookie and a form field, and compare both server-side.
  • Check Referer/Origin Header: Validate incoming requests are from trusted domains.

Best Practices

CSRF protection should be enabled for all state-changing operations (POST, PUT, DELETE). Combine token-based defenses with browser-level controls for stronger protection.
