Core Concepts

Exploit — code that triggers a vulnerability to take control or run code on a target.

Payload — the code delivered by an exploit (e.g., a reverse shell, meterpreter).

  • Module types: exploit, auxiliary, post, payload.
  • Handlers: listeners that accept incoming connections from reverse payloads.
  • Encoders/Listeners: tools to help deliver payloads and avoid detection in lab tests.

Tip: Think of Metasploit as a library of building blocks — pick an exploit, configure options, attach a payload, and run a handler.

Typical Workflow

1. Reconnaissance — discover hosts/services (e.g., nmap) and gather versions.

2. Select module — use Metasploit to search for matching exploits (e.g., `search vsftpd`).

3. Configure — set `RHOST`, `RPORT`, payload options, LHOST for reverse connections:

  • set RHOST 192.168.56.101
  • set LHOST 192.168.56.1

4. Launch & handle — run the exploit and let the handler wait for the payload to connect back.

5. Post‑exploitation — gather data, escalate privileges, and clean up in authorized lab tests.

Safety: Only run these steps in VMs/labs or on systems with explicit permission.
