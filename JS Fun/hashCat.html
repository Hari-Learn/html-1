What is Hashcat?

Hashcat is an advanced password recovery tool used for testing the security of password hashes. It is designed to recover lost passwords or evaluate the strength of existing ones using high-speed processing from CPUs and GPUs.

It is widely used by ethical hackers, cybersecurity researchers, and IT administrators to audit password storage systems. Unauthorized use of Hashcat is illegal and unethical.

Hash Types Supported

  • MD5, SHA-1, SHA-256, SHA-512
  • bcrypt, PBKDF2, scrypt
  • Windows hashes (NTLM), Unix hashes (/etc/shadow)
  • Custom application-specific hashes

Attack Modes (Conceptual)

  • Dictionary: Uses wordlists of known/common passwords.
  • Brute-force: Tries all character combinations.
  • Mask attack: Uses patterns like `??a??n` to reduce guesses.
  • Combinator: Combines two or more words together.
  • Rule-based: Applies smart mutations (e.g., capitalizing, reversing).

Legitimate Use Cases

  • Password recovery for users (with permission)
  • Testing password policy effectiveness
  • Auditing internal systems
  • Educational use in cybersecurity labs

Ethics & Legal Use

Hashcat should only be used in environments where you have explicit permission. Running password recovery on systems or data you don't own is illegal in most countries. Always follow ethical hacking guidelines and local laws.

Defensive Countermeasures

  • Use slow, secure hash algorithms (bcrypt, scrypt, Argon2)
  • Add salts to passwords to prevent hash reuse attacks
  • Use multi-factor authentication (MFA)
  • Educate users on strong, unique passwords
  • Rotate passwords regularly in sensitive systems
