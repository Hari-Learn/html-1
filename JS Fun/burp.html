Burp — Compact Ops

Short notes: installation, automation, OAST, reporting, troubleshooting

Install & License

  • Use bundled Java or compatible JRE.
  • Keep license keys and backups safe.

Headless / CI

  • Run headless JAR for automated scans.
  • Export JSON/XML for CI parsing.

Project Backups

  • Save project files before major changes.
  • Keep exports for team sharing.

Session & Macros

  • Use macros for multi-step logins.
  • Test macros manually before scans.

OAST / Collaborator

  • Use Collaborator to capture blind callbacks.
  • Consider privacy and logs management.

Scan Tuning

  • Adjust speed/threads to limit load.
  • Disable irrelevant checks to reduce noise.

Reporting

  • Export JSON/XML for tracking systems.
  • Obfuscate sensitive data before sharing.

Troubleshoot

  • Install Burp CA if HTTPS missing.
  • Check proxy settings and Java heap for large projects.

APIs & Automation

  • Use Pro API for scripted scans and exports.
  • Rate-limit automation; keep secrets in CI vaults.
