Understanding CSRF (Cross-Site Request Forgery)

CSRF is a web security vulnerability that tricks users into performing unwanted actions on websites where they’re authenticated.

What is CSRF?

Cross-Site Request Forgery occurs when a malicious website causes a user's browser to perform an unwanted action on a trusted site where the user is logged in.

Why is it Dangerous?

If the user is authenticated, the attacker can force actions like changing account settings or making purchases — all without the user's knowledge.

How Does It Work?

Here’s a simple example of how CSRF works: 

<img src="https://example.com/delete-account" />

If you're logged in, the request may go through — and your account might be deleted.

Prevention Techniques

  • CSRF Tokens: Include a unique token in every form.
  • SameSite Cookies: Prevent cookies from being sent on cross-site requests.
  • Referer/Origin Checks: Validate where the request comes from.
  • Double Submit Cookies: Send the CSRF token in both a cookie and a request field.
