Burp Suite — Quick Cheatsheet

Concise summary of the core tools, workflows and shortcuts. Use responsibly — always have permission.

What it is

Burp Suite is a web application security testing platform (PortSwigger). It contains an intercepting proxy plus modular tools for scanning, manipulating and replaying traffic.

Core tools (at-a-glance)

  • Proxy — intercept & modify HTTP(S) requests/responses.
  • Target — site map, manual inspection and issue tracker.
  • Spider / Content discovery — map application (note: active scanning may be noisy).
  • Scanner (Pro) — automated vulnerability scanning.
  • Repeater — craft and resend single requests for manual testing.
  • Intruder — automated fuzzing / parameter-based attacks (careful: can be noisy).
  • Decoder — encode/decode data (Base64, hex, URL, etc.).
  • Comparer — compare responses/requests (diffs).
  • Sequencer — analyze randomness/entropy of tokens.
  • Extender — add BApp extensions (via BApp Store).

Typical quick workflow

  1. Configure browser proxy → 127.0.0.1:8080 and install Burp CA cert for HTTPS.
  2. Proxy → Intercept off (unless capturing specific request). Browse the app to populate Target & Spider.
  3. Review Target site map → identify interesting pages/endpoints.
  4. Send chosen request to Repeater / Intruder / Scanner as needed.
  5. Use Decoder/Comparer for analysis and Extender for automation/plugins.

Safe testing reminder

Only test systems you own or have explicit authorization to test. Unauthorized scanning, fuzzing, or exploitation is illegal and unethical.

Shortcuts & quick tips

Global

  • Ctrl+R — Repeater (send selected request).
  • Ctrl+I — Intruder (send selected request).
  • Ctrl+Shift+I — Open Scanner (Pro) for selected node.
  • Ctrl+P — Proxy settings.

Proxy

  • Intercept: Intercept is On/Off toggle in Proxy tab.
  • Right-click a proxied request → "Send to Repeater/Intruder/Scanner/Comparer".
  • Use "Match and Replace" rules for simple automated modifications.

Common quick actions

  • Save request flow: Project → Save state or Export selected items.
  • Search across history: Proxy → HTTP history → search box.
  • Generate CSRF token patterns: use Sequencer to analyze randomness.
  • Decode an encoded value: Copy → Decoder → select interpretation (Base64, URL, etc.).

Quick tools cheat

  • Proxy: intercept, forward, edit.
  • Repeater: manual request tampering & testing.
  • Intruder: payload positions, payload sets, attack types (Sniper, Clusterbomb, Pitchfork).
  • Scanner: passive & active checks (Pro).
  • Sequencer: entropy, PRNG checks.

Intruder modes (brief)

  • Sniper — one payload position at a time (single param testing).
  • Clusterbomb — multiple positions, combinatorial (powerful & noisy).
  • Pitchfork — parallel payload sets (aligned lengths).
  • Turbo — fastest, less control (use cautiously).

Useful BApp extensions

Popular additions (check BApp Store inside Burp):

  • Logger++ — enhanced logging.
  • JSON Beautifier / Beautifier — pretty-print responses.
  • Autorize — authorization testing helper.
  • Turbo Intruder (separate) — high-performance fuzzing (use with permission).

Common headers & notes

  • Host, User-Agent, Cookie, Referer — often relevant while testing auth/CSRF.
  • Watch for differences in responses: status codes, length, timing, header presence.
  • Use Comparer to diff responses quickly.
