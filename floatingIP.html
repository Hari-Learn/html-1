High Availability & Routing Made Simple

Learn networking concepts by understanding real-world problems and solutions

HSRP

Problem:

If the default gateway router fails, all users lose network connectivity.

Solution:

HSRP creates a virtual gateway shared by two routers. One is active and the other is standby. If the active router fails, the standby router takes over instantly without user impact.

ISP Failover

Problem:

When the primary ISP link goes down, the entire organization loses internet access.

Solution:

ISP Failover automatically switches traffic to a backup ISP using tracking, SLA monitoring, or floating static routes to maintain uninterrupted connectivity.

Floating Static Route

Problem:

Static routes are always preferred and do not automatically act as backups.

Solution:

A floating static route is configured with a higher administrative distance so it only activates when the primary route fails.

Floating IP

Problem:

If a server fails, applications become unreachable because the IP address is tied to that server.

Solution:

A Floating IP moves automatically to a healthy server or device, ensuring continuous access to services.

Administrative Distance

Problem:

Multiple routing protocols may advertise the same destination. Which route should the router trust?

Solution:

Administrative Distance assigns trust levels to routing sources. The route with the lowest AD is selected.

Common AD Values:

  • Connected – 0
  • Static – 1
  • EIGRP – 90
  • OSPF – 110
  • RIP – 120
  • iBGP – 200
  • Unknown – 255
