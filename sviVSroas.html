Switch Virtual Interface (SVI) vs Router-on-a-Stick (ROAS)

Simple Comparison for Interview Explanation — What they can and cannot do

Switch Virtual Interface (SVI)

A virtual Layer-3 interface on a Layer-3 switch that routes between VLANs internally.

  • Routes between VLANs inside the switch (hardware-based, fast)
  • Allows remote switch management via IP (ping, SSH)
  • Scalable solution for large VLAN networks
  • Requires Layer-3 capable switch (not available on pure Layer-2 switches)
  • Cannot be used if switch does not support routing or SVIs

Router-on-a-Stick (ROAS)

Using a single physical router interface with multiple subinterfaces to route between VLANs.

  • Works with Layer-2 switches to enable inter-VLAN routing
  • One router interface supports many VLANs using subinterfaces (802.1Q trunking)
  • Simple and cost-effective for small to medium VLAN routing
  • Routing traffic passes through the router interface → slower than hardware-based SVI routing
  • Requires configuration of subinterfaces and trunking on router and switch

Why do we need both?

SVI is great when your switch supports routing, making inter-VLAN routing fast and easy inside the switch.
ROAS lets you use an external router to connect VLANs when your switches are Layer-2 only.
In many networks, both methods are used depending on hardware and design.
