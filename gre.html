WAN Technologies Moved Beyond CCNA

In newer CCNA versions, some WAN-focused technologies are no longer covered in detail. These topics are now part of higher-level certifications like CCNP or specialized paths.

PPP (Point-to-Point Protocol)

PPP is a WAN protocol used to create a direct connection between two network devices, commonly over serial links.

  • Used in older WAN links
  • Supports authentication (PAP, CHAP)
  • Encapsulates network layer protocols

➜ Detailed configuration is now covered in CCNP.

GRE (Generic Routing Encapsulation)

GRE is a tunneling protocol that allows you to encapsulate one network protocol inside another.

  • Creates virtual point-to-point links
  • Commonly used with VPNs
  • Does NOT provide encryption by itself

➜ Mostly studied with advanced VPN designs.

BGP (Border Gateway Protocol)

BGP is the routing protocol that controls how data is routed between different organizations on the Internet.

  • Used by ISPs and large enterprises
  • Path-vector routing protocol
  • Highly scalable and complex

➜ Full BGP configuration is taught in CCNP/CCIE.
