Routing with OSPF

Open Shortest Path First (OSPF) – Dynamic Routing Protocol

What is OSPF?

OSPF (Open Shortest Path First) is a link-state routing protocol used to find the best path between routers in an IP network. It is commonly used in large enterprise and service provider networks.

Key Features of OSPF

  • Uses Dijkstra’s Shortest Path First (SPF) algorithm
  • Fast convergence
  • Supports VLSM and CIDR
  • Uses cost as the routing metric
  • Classless routing protocol

OSPF Areas

OSPF divides a network into areas to reduce routing overhead and improve performance.

Area 0 (Backbone Area)

All other areas must connect to Area 0. It is the core of the OSPF network.

Non-Backbone Areas

These areas connect to Area 0 and help reduce routing table size.

How OSPF Works

  1. Routers discover neighbors using Hello packets
  2. Routers exchange link-state information
  3. Each router builds a Link-State Database (LSDB)
  4. SPF algorithm calculates the shortest path
  5. Routing table is updated
