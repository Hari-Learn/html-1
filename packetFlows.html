Packet Flow in a Network 🚀

Let’s understand how data travels across a network in a super simple way (ELI10 style).

1. DNS Lookup 🌍

You type a website name like google.com. DNS is like a phonebook — it finds the website’s IP address so your computer knows where to go.

2. TCP Handshake 🤝

Your computer and the server say “Hello” to each other. This 3-step handshake makes sure both sides are ready to talk reliably.

3. Routing Decision 🧭

Routers decide the best path for your data, like Google Maps choosing the fastest route.

4. Switching 🔀

Inside a local network, switches send data to the correct device using MAC addresses.

5. Routing 🌐

Data travels across multiple networks through routers until it gets closer to the destination.

6. Destination Network 🎯

The packet finally reaches the destination network where the target server lives.

7. Decapsulation 📦

The data is unpacked layer by layer (like opening a parcel) until the actual message is revealed.

8. End User 👤

The website loads, and you see the content. Mission complete! 🎉

Simple Flow Summary 🔄

DNS → Handshake → Routing → Switching → Routing → Destination → Decapsulation → You 🎉
