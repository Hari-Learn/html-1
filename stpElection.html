What is STP?

Spanning Tree Protocol (STP) is a Layer 2 network protocol used in switches to prevent network loops. It works by logically blocking redundant paths while keeping one active path between switches.

Root Bridge Election

The Root Bridge is the central reference point in an STP topology. All switches exchange BPDU (Bridge Protocol Data Unit) messages to elect the Root Bridge.

The switch with the lowest Bridge ID becomes the Root Bridge.

What is Bridge ID?

Bridge ID is a unique identifier for each switch and is made of:

  • Bridge Priority
  • MAC Address
Bridge ID = Priority + MAC Address

What is Priority (32768)?

The priority is a configurable value used to influence the Root Bridge election.

  • Default priority value is 32768
  • Valid values are in increments of 4096
  • Lower priority = higher chance to become Root Bridge

If two switches have the same priority, the switch with the lowest MAC address wins.

STP Port Roles

Root Port (RP)

The Root Port is the port on a non-root switch that has the lowest cost path to the Root Bridge. Each non-root switch has only one Root Port.

Designated Port (DP)

A Designated Port is the port that forwards traffic for a network segment. On the Root Bridge, all ports are Designated Ports.

Blocked Port

A Blocked Port does not forward traffic. It is used to prevent loops in the network while still listening for BPDUs.

Quick Summary

  • Root Bridge = Switch with lowest Bridge ID
  • Bridge ID = Priority + MAC Address
  • Default Priority = 32768
  • Root Port = Best path to Root Bridge
  • Designated Port = Forwarding port for a segment
  • Blocked Port = Prevents loops
