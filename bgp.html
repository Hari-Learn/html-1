BGP (Border Gateway Protocol) – CCNA Quick Review

Important concepts you must know for the CCNA exam

What is BGP?

BGP (Border Gateway Protocol) is a path-vector routing protocol used to exchange routing information between different Autonomous Systems (AS).

Why BGP is Used

  • Used on the Internet
  • Handles very large networks
  • Provides better control over routing

Autonomous System (AS)

An Autonomous System is a group of networks controlled by a single organization. Each AS is identified by a unique AS Number (ASN).

eBGP vs iBGP

eBGP Between different AS
iBGP Within the same AS

BGP Path Selection (Important)

BGP does NOT choose shortest path. It chooses the best path based on attributes.

  • Highest Weight (Cisco only)
  • Highest Local Preference
  • Shortest AS Path
  • Lowest Origin Type

Important BGP Attributes

  • Weight
  • Local Preference
  • AS Path
  • Origin
  • Next Hop

Transport & Port

BGP uses TCP as its transport protocol and operates on Port 179.

BGP Characteristics

  • Classless protocol
  • Incremental updates
  • Highly scalable
  • Policy-based routing

CCNA Exam Tips

  • Know eBGP vs iBGP difference
  • Remember TCP port 179
  • Understand AS Path concept
  • BGP ≠ IGP (OSPF, RIP)
