Half-Duplex vs Full-Duplex Behavior

Half-Duplex

In half-duplex communication, data can be transmitted in both directions, but not at the same time.

  • Only one device can transmit at a time
  • Collisions can occur
  • Uses CSMA/CD for collision detection
  • Common in hub-based networks

Half-Duplex Example

Walkie-talkies are a classic example of half-duplex communication — only one person can talk at a time while others listen.

Full-Duplex

In full-duplex communication, data can be transmitted and received simultaneously between devices.

  • Both devices transmit at the same time
  • No collisions occur
  • CSMA/CD is not required
  • Used by modern Ethernet switches

Full-Duplex Example

Phone calls represent full-duplex communication — both people can speak and listen at the same time.

Key Comparison

Half-Duplex: One direction at a time, collisions possible.

Full-Duplex: Simultaneous two-way communication, no collisions.
