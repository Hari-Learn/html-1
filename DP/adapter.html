Adapter Design Pattern

Converts one interface into another interface.

What is Adapter Pattern?

The Adapter Design Pattern allows two incompatible interfaces to work together. It acts like a bridge between existing classes by converting one interface into another expected interface.

Benefits

✅ Reuses existing code
✅ Connects incompatible systems
✅ Improves flexibility
✅ Reduces code changes

Java Example 


interface MediaPlayer {
    void play(String audioType);
}

class Mp3Player {
    public void playMp3() {
        System.out.println("Playing MP3");
    }
}

class Adapter implements MediaPlayer {

    private Mp3Player mp3Player;

    Adapter(Mp3Player mp3Player) {
        this.mp3Player = mp3Player;
    }

    public void play(String audioType) {
        mp3Player.playMp3();
    }
}

public class Main {
    public static void main(String[] args) {

        MediaPlayer player =
            new Adapter(new Mp3Player());

        player.play("mp3");
    }
}

Real World Example

A mobile charger adapter is a real-world example. The adapter converts electrical connections so different devices can work with the same power socket.
