Future / Promise Design Pattern

The Future/Promise pattern is used to handle asynchronous operations. Instead of waiting for a task to complete, a placeholder object (Future or Promise) is returned immediately. The result can be accessed later when the task finishes.

Real-World Example

Ordering food online: after placing an order, you receive a token (Promise). You continue doing other things while the restaurant prepares the food. Later, the order is delivered and the Promise is fulfilled.

Key Benefits

  • Improves application responsiveness.
  • Prevents blocking while waiting for long-running tasks.
  • Simplifies asynchronous programming.
  • Supports chaining and error handling.
// JavaScript Promise Example

function fetchData() {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve("Data Loaded");
    }, 2000);
  });
}

fetchData()
  .then(result => console.log(result))
  .catch(error => console.error(error));
Pattern Category: Concurrency / Asynchronous Programming
