🗄️ Cache-Aside Pattern

The Cache-Aside pattern, also known as Lazy Caching, loads data into the cache only when it is requested. If the data is not in the cache, it is fetched from the database, stored in the cache, and then returned.

How It Works

  1. The application requests data.
  2. Check if the data exists in the cache.
  3. If it's a cache miss, read from the database.
  4. Store the data in the cache and return it.

Pseudo Code

data = cache.get(key);

if (data == null) {
    data = database.get(key);
    cache.put(key, data);
}

return data;

Commonly used with Redis, Memcached, and distributed caching systems to improve read performance while reducing database load.
