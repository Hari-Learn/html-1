Correlation Identifier

The Correlation Identifier pattern assigns a unique ID to a request so it can be tracked across multiple services in a distributed system.

Client Request
↓
Service A (ID: 123)
↓
Service B (ID: 123)
↓
Service C (ID: 123)
↓
Response Tracked End-to-End
  • Helps trace requests across services.
  • Improves debugging in microservices.
  • Enables end-to-end observability.
  • Often used with logging and tracing systems.

Example: A user request in a payment system carries the same Correlation ID through order service, payment service, and notification service.
