CQRS (Command Query Responsibility Segregation)

CQRS is an architectural pattern that separates operations that modify data (Commands) from operations that read data (Queries). Instead of using the same model for both reading and writing, CQRS uses different models and responsibilities for each.

Core Concepts

  • Command – Creates, updates, or deletes data.
  • Query – Retrieves data without changing it.
  • Read and write operations are handled separately.
  • Improves scalability and maintainability in complex systems.

Benefits

  • Clear separation of responsibilities.
  • Optimized read and write models.
  • Easier scaling of large applications.
  • Better maintainability and flexibility.

Simple Example 

class UserCommandService {
    createUser(name) {
        console.log(`User created: ${name}`);
    }
}

class UserQueryService {
    getUser(id) {
        return { id, name: "John Doe" };
    }
}

// Command (Write)
const commandService = new UserCommandService();
commandService.createUser("John Doe");

// Query (Read)
const queryService = new UserQueryService();
console.log(queryService.getUser(1));

In this example, UserCommandService is responsible for modifying data, while UserQueryService is responsible for retrieving data. This separation is the core idea behind CQRS.
