Private Class Data Pattern

Separates implementation data from the public interface to improve encapsulation.

What is Private Class Data?

The Private Class Data Pattern moves internal data into a separate private object. The main class exposes only required operations, preventing direct access and accidental modification of internal state.

Example 

class BankAccountData {
  constructor(balance) {
    this.balance = balance;
  }
}

class BankAccount {

  constructor(balance) {
    this.privateData = new BankAccountData(balance);
  }

  getBalance() {
    return this.privateData.balance;
  }

  deposit(amount) {
    this.privateData.balance += amount;
  }
}

const account = new BankAccount(1000);

account.deposit(500);

console.log(account.getBalance());
// 1500

Key Idea

  • Keeps internal data hidden from external access.
  • Improves encapsulation and protects object state.
  • Allows implementation details to change without affecting users.
  • Useful for complex objects with sensitive internal information.
