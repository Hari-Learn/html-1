Creational Design Pattern

Singleton Design Pattern

Ensures a class has only one instance and provides a global point of access to it.

What is Singleton?

The Singleton Pattern restricts object creation so that only one instance of a class exists throughout the application lifecycle.

This pattern is commonly used when exactly one object is needed to coordinate actions across a system.

Single Instance

Only one object can exist.

Global Access

Accessible from anywhere in the application.

Lazy Initialization

Instance can be created only when required.

JavaScript Example 

class Singleton {
  static instance;

  constructor() {
    if (Singleton.instance) {
      return Singleton.instance;
    }

    this.data = "Single Instance Data";
    Singleton.instance = this;
  }
}

const obj1 = new Singleton();
const obj2 = new Singleton();

console.log(obj1 === obj2); // true

Real World Applications

🗄 Database Connection

Prevents multiple database connections and reduces resource usage.

⚙ Configuration Manager

Stores application-wide configuration settings.

📝 Logger Service

Centralized logging across the application.

🌐 State Management

Maintains a single source of truth for application state.

Advantages

  • ✅ Controlled access to a single instance
  • ✅ Saves memory and resources
  • ✅ Global access point
  • ✅ Useful for shared services

Disadvantages

  • ❌ Difficult to unit test
  • ❌ Can introduce global state
  • ❌ Hidden dependencies
  • ❌ May violate Single Responsibility Principle
