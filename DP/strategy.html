Strategy Design Pattern

Encapsulates interchangeable algorithms.

What is Strategy Pattern?

The Strategy Pattern defines a family of algorithms, places each one in a separate class, and makes them interchangeable at runtime.

Real World Example

Google Maps

Choose Car, Bike, or Walking routes.

Payment System

Pay using Card, UPI, or PayPal.

Sorting

Switch between Bubble, Quick, or Merge sort.

Pattern Flow

  1. 1 Define multiple algorithms separately.
  2. 2 Context object uses one strategy at a time.
  3. 3 Strategy can change dynamically.
  4. 4 Different behaviors are achieved without modifying the main class.

Simple Example 

class PaymentStrategy {
  pay(amount) {}
}

class CardPayment extends PaymentStrategy {
  pay(amount) {
    console.log("Paid using Card:", amount);
  }
}

class UpiPayment extends PaymentStrategy {
  pay(amount) {
    console.log("Paid using UPI:", amount);
  }
}
