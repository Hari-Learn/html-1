Init Container

An Init Container is a special container that runs before the main application container starts. It performs setup tasks and exits successfully before the application begins running.

Start Pod
↓
Init Container
(Setup / Configuration)
↓
Main Application Container
↓
Application Running
  • Prepares configuration files.
  • Waits for dependencies to become available.
  • Initializes databases or resources.
  • Keeps startup logic separate from the main app.

Example: An init container downloads configuration files from a remote server before the main application container starts.
