Chain of Responsibility

The Chain of Responsibility design pattern passes a request through a chain of handlers. Each handler decides whether to process the request or forward it to the next handler.

Request
→
Handler 1
→
Handler 2
→
Handler 3

Example

Imagine a customer support system:

  • Level 1 handles basic issues
  • Level 2 handles technical issues
  • Manager handles critical issues

If one handler cannot process the request, it forwards it to the next handler.
