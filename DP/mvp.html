MVP (Model–View–Presenter)

MVP (Model–View–Presenter) is an architectural pattern that separates an application into three components: Model, View, and Presenter. The Presenter acts as a middle layer between the View and the Model, handling user interactions and updating the View.

Components of MVP

  • Model – Manages data and business logic.
  • View – Displays data and captures user input.
  • Presenter – Connects the Model and View.

Benefits

  • Clear separation of concerns.
  • Improves maintainability.
  • Easier unit testing of business logic.
  • Reduces dependency between UI and data layers.

Simple Example 

class UserModel {
    getUser() {
        return { name: "John Doe" };
    }
}

class UserView {
    displayUser(user) {
        console.log("User:", user.name);
    }
}

class UserPresenter {
    constructor(model, view) {
        this.model = model;
        this.view = view;
    }

    loadUser() {
        const user = this.model.getUser();
        this.view.displayUser(user);
    }
}

const presenter = new UserPresenter(
    new UserModel(),
    new UserView()
);

presenter.loadUser();

In this example, the Model provides user data, the View displays it, and the Presenter coordinates communication between them. This keeps the UI logic separate from the business logic.
