Claim Check Pattern

The Claim Check pattern stores large payloads outside the message and sends only a reference (claim check) through the messaging system.

This keeps messages small and improves performance when handling large files or documents.

How It Works

  1. Store the large payload in external storage.
  2. Generate a reference ID.
  3. Send the reference in the message.
  4. Receiver uses the reference to retrieve the payload.

Benefits

  • Smaller message sizes.
  • Faster message processing.
  • Reduced network usage.
  • Better scalability.

Simple Flow

Large File
↓
External Storage
↓
Reference ID
↓
Message Queue
↓
Consumer Retrieves File
