🚩 Dirty Flag Pattern

The Dirty Flag pattern delays expensive calculations until they are actually needed. When data changes, it is marked as "dirty". The value is recalculated only when someone requests it.

How It Works

  1. Data changes.
  2. Set the dirty flag to true.
  3. When the value is needed, check the flag.
  4. If dirty, recalculate the value.
  5. Clear the flag and return the updated result.

Pseudo Code

if (dirty) {
    recalculate();
    dirty = false;
}

return cachedValue;

Commonly used in game engines, graphics rendering, scene graphs, caching, and UI frameworks to avoid unnecessary computations.
