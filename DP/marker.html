Marker Pattern

A design pattern that uses an empty interface or class to mark objects with special behavior or capabilities.

The Problem

Sometimes we need to identify certain objects as having special characteristics without adding methods or properties. Using type checks or flags throughout the code can become difficult to manage.

The Solution

Create an empty interface (marker interface). Classes that implement it are automatically recognized as belonging to a special category. 

interface Serializable {
    // No methods
}

class User implements Serializable {
}

How It Works

User Object
→
Implements Marker Interface
→
Special Handling

Real-World Example

Java's Serializable interface is one of the most famous marker interfaces. 

if (object instanceof Serializable) {
    saveToFile(object);
}

Common Use Cases

  • ✓ Serialization support
  • ✓ Security and authorization
  • ✓ Framework-specific annotations
  • ✓ Special processing rules
  • ✓ Metadata tagging

Advantages

  • ✔ Simple to implement
  • ✔ Easy object categorization
  • ✔ No extra methods required
  • ✔ Works well with frameworks

Challenges

  • ✖ Behavior is not explicit
  • ✖ Can reduce code clarity
  • ✖ Modern annotations often replace it
  • ✖ May lead to misuse of type checks

Key Takeaway

The Marker Pattern uses empty interfaces or classes to tag objects with special meaning. It provides a lightweight way to identify capabilities or categories without adding extra behavior directly to the interface.
