Composite Design Pattern

The Composite Design Pattern is a structural design pattern used to treat individual objects and groups of objects in the same way.

Definition:
“Treat groups and individual objects uniformly.”

Why Use Composite Pattern?

  • Simplifies working with tree-like structures.
  • Allows clients to treat single and grouped objects equally.
  • Makes recursive structures easier to manage.

Example

A file system where both files and folders are treated as items.

Folder → File / Folder

Benefits

  • Cleaner hierarchical code structure
  • Easier scalability
  • Improves flexibility and maintainability
