MVVM (Model–View–ViewModel)

MVVM (Model–View–ViewModel) is an architectural pattern that separates the user interface from business logic. The ViewModel acts as a bridge between the Model and the View, exposing data and commands that the View can bind to.

Components of MVVM

  • Model – Contains data and business rules.
  • View – Displays data to the user.
  • ViewModel – Prepares and manages data for the View.

Benefits

  • Clear separation between UI and business logic.
  • Supports data binding.
  • Improves maintainability and scalability.
  • Makes testing easier.

Simple Example 

class UserModel {
    getUser() {
        return { name: "John Doe" };
    }
}

class UserViewModel {
    constructor(model) {
        this.model = model;
    }

    getUserName() {
        return this.model.getUser().name;
    }
}

class UserView {
    render(viewModel) {
        console.log("User:", viewModel.getUserName());
    }
}

const model = new UserModel();
const viewModel = new UserViewModel(model);
const view = new UserView();

view.render(viewModel);

In this example, the Model stores user data, the ViewModel prepares the data for display, and the View renders it. The View does not directly communicate with the Model, which helps keep concerns separated.
