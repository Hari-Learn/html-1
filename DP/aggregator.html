Aggregator Pattern

A design pattern that collects data from multiple services, systems, or sources and combines it into a single response for the client.

The Problem

In a microservices architecture, a client may need data from several services to display a single screen.

Without an Aggregator:

  • Client calls User Service
  • Client calls Order Service
  • Client calls Payment Service
  • Client combines responses

This increases network calls and adds complexity to the client.

The Solution

Introduce an Aggregator service that gathers information from multiple services and returns a unified response.

Client
→
Aggregator
→
User Service
Order Service
Payment Service

Example Response 

{
  "user": {
    "name": "John"
  },
  "orders": [
    {
      "id": 101
    }
  ],
  "payments": {
    "status": "Paid"
  }
}

The client receives all required information through a single API call.

Workflow

  1. 1. Client sends a request.
  2. 2. Aggregator receives the request.
  3. 3. Aggregator calls multiple services.
  4. 4. Responses are collected.
  5. 5. Data is merged into a unified model.
  6. 6. Aggregated response is returned.

Common Use Cases

👤 User Dashboard

Combine profile, orders, and notifications.

🛒 E-Commerce

Aggregate product, inventory, and pricing data.

📱 Mobile Apps

Reduce API calls and improve performance.

☁️ Microservices

Present a unified API over many services.

Advantages

  • ✔ Fewer client requests
  • ✔ Simplified client code
  • ✔ Better user experience
  • ✔ Centralized data composition
  • ✔ Reduced network overhead

Challenges

  • ✖ Additional service to maintain
  • ✖ Potential performance bottleneck
  • ✖ Increased complexity on the server side
  • ✖ Error handling across multiple services
  • ✖ Dependency on downstream services

Key Takeaway

The Aggregator Pattern acts as a central point that gathers information from multiple services and delivers a single, consolidated response. It is commonly used in microservices architectures to simplify clients and reduce network traffic.
