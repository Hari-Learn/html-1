Pimpl (Pointer to Implementation)

A C++ design technique used to reduce compile-time dependencies and hide implementation details.

What is Pimpl?

Pimpl stands for Pointer to Implementation . Instead of storing all member variables directly inside a class, the class stores only a pointer to a hidden implementation class.

Goal: Hide implementation details and reduce header file dependencies.

Why Use Pimpl?

✅ Faster Compilation

Changing implementation details does not force every source file to recompile.

🔒 Information Hiding

Private data members remain hidden from users of the class.

📦 Stable Interface

Public header files rarely change even when implementation changes frequently.

⚙️ Better Build Times

Large projects compile much faster because fewer files need rebuilding.

Structure

Widget
 ├── Widget.h
 │      class Widget
 │      {
 │          Impl* impl;
 │      };
 │
 └── Widget.cpp
        class Widget::Impl
        {
            // Hidden implementation
        };

Advantages

  • Reduces compile-time dependencies.
  • Improves encapsulation.
  • Reduces header file changes.
  • Helps maintain binary compatibility.
  • Ideal for large C++ projects and libraries.

Disadvantages

  • Extra heap allocation.
  • Additional pointer dereference.
  • Slight runtime overhead.
  • More boilerplate code.

ELI10 Example

Imagine you have a TV remote. You only use the buttons on the outside. You don't need to know the electronics inside the remote.

With Pimpl:

  • The remote = Public class (Header file)
  • The electronics inside = Hidden implementation (.cpp file)
  • You can redesign the electronics without changing the buttons.
Key Idea: Users only see the interface. The implementation stays hidden, resulting in fewer recompilations and cleaner code.
