Circuit Breaker Pattern

Prevent cascading failures by stopping requests to unhealthy services.

What is Circuit Breaker?

The Circuit Breaker Pattern monitors service calls and temporarily blocks requests when failures exceed a threshold. This prevents overloaded or failing services from causing widespread system issues.

Circuit States

Closed

Requests flow normally.

Half-Open

Limited requests are allowed for testing.

Open

Requests are blocked until recovery.

How It Works

Client Request
↓
Service Failure Detected
↓
Failure Threshold Reached
↓
Circuit Opens
↓
Requests Rejected Temporarily
↓
Recovery Test (Half-Open)
↓
Circuit Closes if Successful

Example

An Order Service depends on a Payment Service. If the Payment Service starts timing out repeatedly, the circuit breaker opens and immediately returns a fallback response instead of waiting for more failed calls.

Benefits

  • Prevents cascading failures
  • Improves system resilience
  • Reduces response latency during outages
  • Allows failing services time to recover
  • Provides better user experience with fallbacks
