📄 Copy-on-Write (COW) Pattern

The Copy-on-Write pattern allows multiple objects to share the same data until one of them needs to modify it. A new copy is created only when a write operation occurs, saving memory and improving performance.

How It Works

  1. Multiple objects share the same data.
  2. Read operations use the shared data directly.
  3. When an object wants to modify the data, create a private copy.
  4. Update the copied data without affecting others.

Pseudo Code

if (shared) {
    data = copy(data);
}

modify(data);

Commonly used in operating systems, memory management, immutable data structures, databases, and frameworks where many reads occur but writes are relatively infrequent.
