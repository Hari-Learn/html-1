Abstract Factory Design Pattern

Creates families of related objects without specifying their concrete classes.

What is Abstract Factory?

The Abstract Factory Pattern provides an interface for creating related or dependent objects without directly specifying their classes.

It is commonly used when:

  • Multiple related objects are needed
  • You want consistent object families
  • The system should be independent of object creation
  • Supporting multiple themes or platforms

JavaScript Example 

class WindowsButton {
  paint() {
    return "Windows Button";
  }
}

class MacButton {
  paint() {
    return "Mac Button";
  }
}

class GUIFactory {
  static createButton(type) {
    if (type === "windows") {
      return new WindowsButton();
    }

    if (type === "mac") {
      return new MacButton();
    }
  }
}

const button = GUIFactory.createButton("windows");

console.log(button.paint());

Advantages

✔ Ensures compatible objects
✔ Improves consistency
✔ Reduces tight coupling
✔ Easy to extend product families
