🏖️ Subclass Sandbox Pattern

The Subclass Sandbox pattern defines a fixed framework in a base class while allowing subclasses to customize specific behavior using protected helper methods. This gives subclasses flexibility without exposing the entire system.

How It Works

  1. Create a base class with a common workflow.
  2. Provide protected helper methods.
  3. Subclasses override only the required behavior.
  4. The base class controls the overall execution.

Pseudo Code

class Enemy {

    update() {
        move();
        attack();
    }

}

class Dragon extends Enemy {

    attack() {
        breatheFire();
    }

}

Commonly used in game engines, frameworks, and libraries where a base class provides the overall algorithm while subclasses implement game-specific or application-specific behavior.
