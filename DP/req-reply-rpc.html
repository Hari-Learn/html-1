📨 Request-Reply Pattern

The Request-Reply pattern allows one service to send a request message and wait for a response message. It is commonly used to implement RPC (Remote Procedure Call) over messaging, enabling communication between distributed systems.

How It Works

  1. A client sends a request message.
  2. The message broker delivers it to a service.
  3. The service processes the request.
  4. The service sends a reply back to the client.

Pseudo Code

client.send(request);

response = client.waitForReply();

process(response);

Commonly used in RabbitMQ, Apache Kafka, Azure Service Bus, gRPC, and enterprise messaging systems where services communicate asynchronously while still expecting a response.
