1

Prepare Phase

  • • Coordinator asks all participants to prepare.
  • • Each participant validates the transaction.
  • • Replies with YES or NO.
2

Commit Phase

  • • If everyone votes YES, commit.
  • • If any participant votes NO, rollback.
  • • All participants reach the same final state.

Transaction Flow 

Coordinator
      │
      ├── PREPARE ─────────► Participants
      │◄──── YES / NO ──────
      │
      ├── COMMIT (All YES)
      │          OR
      └── ROLLBACK (Any NO)

Key Idea

Two-Phase Commit (2PC) guarantees that a distributed transaction is atomic—either every participant commits the transaction or every participant rolls it back, ensuring consistency across all systems.
