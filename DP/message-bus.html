🚌 Message Bus Pattern

The Message Bus pattern provides a centralized communication channel through which applications or services exchange messages. Components communicate through the bus instead of talking directly to each other, reducing coupling.

How It Works

  1. A sender publishes a message to the bus.
  2. The Message Bus receives the message.
  3. The bus routes the message to the appropriate receiver(s).
  4. Subscribers process the received message.

Pseudo Code

messageBus.publish(message);

subscriber.onMessage(message);

process(message);

Commonly used in enterprise applications, microservices, event-driven systems, CQRS architectures, and messaging platforms to enable scalable, centralized communication.
