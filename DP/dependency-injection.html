Dependency Injection (DI)

Dependency Injection (DI) is a design pattern used to implement Inversion of Control (IoC). Instead of a class creating its own dependencies, the dependencies are provided from outside.

Benefits of Dependency Injection

  • Reduces tight coupling between classes.
  • Improves code maintainability.
  • Makes unit testing easier.
  • Promotes flexibility and reusability.

Simple Example

class EmailService {
    send(message) {
        console.log(message);
    }
}

class Notification {
    constructor(emailService) {
        this.emailService = emailService;
    }

    notify(message) {
        this.emailService.send(message);
    }
}

const emailService = new EmailService();
const notification = new Notification(emailService);

notification.notify("Hello DI!");

In this example, the Notification class does not create an EmailService object itself. Instead, the dependency is injected through the constructor.
