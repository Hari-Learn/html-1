🔁 Idempotency Pattern

The Idempotency pattern ensures that performing the same operation multiple times produces the same result as performing it once. This makes retries safe when requests are duplicated due to network or system failures.

How It Works

  1. A client sends a request with a unique idempotency key.
  2. The server checks if the key has already been processed.
  3. If it is new, process the request and store the result.
  4. If it already exists, return the previous result.

Pseudo Code

if (processed(key)) {
    return previousResult;
}

result = processRequest();
save(key, result);

return result;

Commonly used in payment systems, REST APIs, distributed systems, microservices, and message processing to safely handle retries without creating duplicate operations.
