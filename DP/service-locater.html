Service Locator Design Pattern

A centralized registry that provides access to application services.

What is Service Locator?

Service Locator is a design pattern that hides the process of locating and obtaining services. Instead of creating dependencies directly, objects request required services from a central Service Locator.

How It Works

Client

Requests Service

→
Service Locator

Finds / Caches Service

→
Service

Provides Functionality

Advantages

  • Centralized service management
  • Reduces object creation complexity
  • Supports service caching
  • Improves code reuse

Disadvantages

  • Hidden dependencies
  • Harder to unit test
  • Can increase coupling
  • Less explicit than Dependency Injection

Simple Example

Client → ServiceLocator.getService("EmailService")
ServiceLocator → Returns EmailService Instance
Client → emailService.sendMessage()

The client does not create the Email Service directly. Instead, it asks the Service Locator to provide the required service.

Service Locator simplifies service access through a centralized registry.
