Multiton Pattern

Similar to Singleton but allows a fixed number of named instances.

What is Multiton?

The Multiton Pattern restricts the creation of objects by maintaining a collection of named instances. Instead of allowing only one instance like Singleton, it allows a predefined number of unique instances.

Example 

class DatabaseMultiton {
  static instances = {};

  static getInstance(name) {
    if (!this.instances[name]) {
      this.instances[name] = new DatabaseMultiton(name);
    }

    return this.instances[name];
  }

  constructor(name) {
    this.name = name;
  }
}

const db1 = DatabaseMultiton.getInstance("users");
const db2 = DatabaseMultiton.getInstance("orders");

console.log(db1.name); // users
console.log(db2.name); // orders

Key Idea

  • Singleton → Only one object exists.
  • Multiton → Multiple controlled objects exist.
  • Each instance is identified by a unique name/key.
  • Useful for managing resources like database connections, caches, or configurations.
