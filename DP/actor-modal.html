Actor Model Pattern

The Actor Model is a concurrency pattern where independent units called Actors communicate exclusively through messages. Each actor has its own state and processes messages one at a time, eliminating the need for shared memory and reducing concurrency issues.

Message Flow

Actor A
➜ Message ➜
Actor B
➜ Message ➜
Actor C

Core Principles

  • Actors communicate only through asynchronous messages.
  • Each actor manages its own private state.
  • No shared mutable memory between actors.
  • Actors can create other actors.
  • Actors decide how to handle incoming messages.

Real-World Example

A food delivery platform can model restaurants, delivery partners, and customers as separate actors. Each actor exchanges messages such as "Order Received", "Food Prepared", and "Order Delivered" without directly accessing another actor's internal state.

Benefits

  • Simplifies concurrent programming.
  • Avoids race conditions and locking complexities.
  • Highly scalable and fault tolerant.
  • Works well in distributed systems.
// Simple Actor Example (Pseudo Code)

class Actor {
  receive(message) {
    console.log("Received:", message);
  }
}

const actorA = new Actor();
const actorB = new Actor();

// Send message
actorB.receive("Hello from Actor A");
