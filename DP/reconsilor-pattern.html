Reconciler Pattern

The Reconciler Pattern continuously compares the current state of a system with the desired state and takes actions to eliminate any differences.

Desired State
⇅
Reconciler
(Compare & Fix)
⇅
Current State
  • Detects configuration drift.
  • Automatically restores the desired state.
  • Improves reliability and consistency.
  • Common in Kubernetes controllers and operators.

Example: If a deployment should have 3 running pods but only 2 exist, the reconciler creates the missing pod to match the desired state.
