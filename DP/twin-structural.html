Twin Pattern

Category: Structural Design Pattern

Split one logical object into two cooperating classes.

Overview

The Twin Pattern divides a single logical entity into two closely connected classes. Each class handles a specific responsibility while collaborating with the other, allowing developers to overcome language limitations such as single inheritance and improve code organization.

Purpose

  • Separate responsibilities into two classes.
  • Allow both classes to work as one logical object.
  • Improve maintainability and readability.
  • Handle multiple behaviors without multiple inheritance.

How It Works

  1. Create two related classes.
  2. Each class stores a reference to its twin.
  3. Responsibilities are divided between them.
  4. They collaborate to provide complete functionality.

Simple Example

class GameLogic {
    Animation animation;

    void move() {
        System.out.println("Moving...");
        animation.play();
    }
}

class Animation {
    GameLogic logic;

    void play() {
        System.out.println("Playing animation");
    }
}

In this example, GameLogic handles movement while Animation handles visual effects. Together they behave as one logical object.

Advantages

  • Better separation of concerns.
  • Cleaner and modular code.
  • Avoids multiple inheritance issues.
  • Easier testing and maintenance.

Disadvantages

  • Requires synchronization between twins.
  • Extra references increase complexity.
  • Not suitable for simple objects.
  • Can be harder to understand initially.
