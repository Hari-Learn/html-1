DAO (Data Access Object)

DAO is a design pattern that separates data access logic from business logic. It provides a dedicated layer for interacting with a database.

Benefits

  • Keeps business code clean.
  • Makes database changes easier to manage.
  • Improves maintainability and testability.
Example: A UserDAO class handles creating, reading, updating, and deleting user records without exposing database details to the application.
