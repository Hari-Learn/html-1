🛡️ Policy Enforcement Point (PEP)

The Policy Enforcement Point (PEP) is responsible for enforcing authorization decisions. Before allowing access to a protected resource, it checks whether the user has the required permissions.

How It Works

  1. A user requests access to a resource.
  2. The PEP intercepts the request.
  3. The PEP asks the authorization service for a decision.
  4. If access is allowed, the request continues; otherwise, it is denied.

Pseudo Code

if (authorize(user, resource)) {
    allowAccess();
} else {
    denyAccess();
}

Commonly used in API gateways, web applications, enterprise security, OAuth, OpenID Connect, and role-based or attribute-based access control systems.
