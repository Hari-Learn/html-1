Scatter-Gather Pattern

Scatter-Gather is an integration and distributed systems pattern where a request is sent to multiple services in parallel (scatter), and the responses are collected and combined (gather) into a single result.

This pattern improves response time by executing multiple operations concurrently instead of sequentially.

How It Works

  1. A client sends a request.
  2. The request is distributed to multiple services.
  3. Services process the request independently.
  4. Responses are collected from all services.
  5. The results are aggregated into a single response.

Benefits

  • Faster response times through parallel processing.
  • Efficient use of distributed systems.
  • Supports aggregation from multiple data sources.
  • Improves scalability and flexibility.

Common Use Cases

  • Travel booking systems comparing flight prices.
  • E-commerce product search across vendors.
  • Microservices-based dashboards.
  • Real-time analytics and reporting systems.

Example

A travel website sends the same search request to multiple airline providers simultaneously. Once all responses arrive, it combines them into a single list of available flights.
