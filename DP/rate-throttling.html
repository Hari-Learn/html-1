🚦 Rate Limiting Pattern

The Rate Limiting pattern controls how many requests a user or client can make within a specific time period. If the limit is exceeded, additional requests are delayed or rejected. This is also known as request throttling.

How It Works

  1. Receive a client request.
  2. Check how many requests were made recently.
  3. If under the limit, process the request.
  4. If the limit is exceeded, reject or delay the request.

Pseudo Code

if (requestCount < LIMIT) {
    allowRequest();
} else {
    rejectRequest(); // HTTP 429
}

Commonly used in APIs, web servers, cloud services, and microservices to prevent abuse, protect resources, and ensure fair usage.
