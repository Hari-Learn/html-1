Multiton Design Pattern

The Multiton pattern is similar to Singleton, but instead of having only one instance, it maintains one instance per key.

Example:
  • One database connection per tenant
  • One logger per application module
  • One cache instance per region
Multiton.get("US")
Multiton.get("EU")
Multiton.get("US")

// Same US instance reused

Key Idea: One shared instance for each unique key.
