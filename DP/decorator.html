Decorator Design Pattern

The Decorator Design Pattern is a structural design pattern used to add new behavior to objects dynamically without changing their original code.

Definition:
“Add behavior dynamically.”

Why Use Decorator Pattern?

  • Add features without modifying existing classes.
  • Supports flexible and reusable code.
  • Follows the Open/Closed Principle.

Example

A coffee ordering system where extra toppings like milk or chocolate can be added dynamically.

Coffee → Milk → Chocolate

Benefits

  • Flexible feature extension
  • Avoids large inheritance hierarchies
  • Improves maintainability
