DTO (Data Transfer Object)

DTO is a design pattern used to transfer data between different layers or systems. It contains only data and does not include business logic.

Benefits

  • Reduces the number of remote calls.
  • Improves data transfer efficiency.
  • Separates internal models from external APIs.
Example: A UserDTO object may contain only id, name, and email fields that are sent from the server to the client.
