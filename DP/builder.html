Builder Design Pattern

A Creational Design Pattern used to construct complex objects step by step.

What is Builder Pattern?

The Builder Pattern helps create complex objects using a step-by-step approach. Instead of using a large constructor with many parameters, the builder pattern allows flexible and readable object creation.

Advantages

✅ Improves code readability
✅ Handles complex object creation
✅ Avoids telescoping constructors
✅ Easy to maintain and extend

Java Example


class Car {
    private String engine;
    private int wheels;

    public static class Builder {
        private String engine;
        private int wheels;

        public Builder setEngine(String engine) {
            this.engine = engine;
            return this;
        }

        public Builder setWheels(int wheels) {
            this.wheels = wheels;
            return this;
        }

        public Car build() {
            Car car = new Car();
            car.engine = this.engine;
            car.wheels = this.wheels;
            return car;
        }
    }
}

Real World Example

Ordering a burger in a restaurant is similar to the Builder Pattern. You can customize ingredients step by step:

  • Select bun type
  • Add cheese
  • Add vegetables
  • Choose sauces
