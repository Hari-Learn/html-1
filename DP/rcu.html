Read-Copy-Update (RCU)

Read-Copy-Update (RCU) is a concurrency pattern that allows many threads to read shared data without locks while updates happen safely in the background.

Readers access the current version of data. Writers create a copy, apply changes, and then replace the old version with the new one.

How It Works

  1. Readers access shared data without locking.
  2. A writer creates a copy of the data.
  3. The writer updates the copied version.
  4. The new version replaces the old version.
  5. The old version is removed after all readers finish.

Benefits

  • Very fast read operations.
  • Minimal lock contention.
  • Scales well for read-heavy workloads.
  • Commonly used in operating system kernels.

Simple Flow

Readers → Current Data
↑
Old Version ← Replace ← Updated Copy
↑
Writer Creates Copy
