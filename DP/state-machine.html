🔄 State Machine Pattern

The State Machine pattern is commonly used in lexical analysis (lexing). It processes input one character at a time, changing states to recognize tokens such as identifiers, numbers, keywords, and operators.

How It Works

  1. Start in the initial state.
  2. Read one character from the input.
  3. Transition to the next state based on the character.
  4. When a token is complete, emit it and continue scanning.

Pseudo Code

state = START;

while (hasInput()) {
    state = nextState(state, readChar());

    if (isTokenComplete(state)) {
        emitToken();
    }
}

Commonly used in compilers, interpreters, regular expression engines, protocol parsers, and text-processing tools for efficient lexical analysis.
