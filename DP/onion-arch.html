Onion Architecture

Onion Architecture organizes software into concentric layers, placing business rules at the center and keeping external concerns at the outer layers.

Infrastructure Layer
Application Layer
Domain Services
Core Domain

Core Domain

Contains entities, business rules, and domain logic. This layer should not depend on any other layer.

Domain Services

Encapsulates business operations that don't naturally belong to a single entity.

Application Layer

Coordinates workflows, use cases, and communication between domain and external systems.

Infrastructure Layer

Handles databases, APIs, file systems, messaging, and other external dependencies.

Key Principle

Dependencies always point inward. Outer layers can depend on inner layers, but inner layers must never depend on outer layers.
