Strangler Fig Pattern

Modernize legacy systems incrementally without a risky full rewrite.

What is the Strangler Fig Pattern?

The Strangler Fig Pattern is a software modernization strategy where a legacy application is gradually replaced by a new system. Instead of rebuilding everything at once, functionality is migrated piece by piece until the old system can be safely retired.

Why is it Called "Strangler Fig"?

The pattern is inspired by the strangler fig tree, which grows around an existing tree. Over time, it gradually replaces the original tree. Similarly, new software components grow around a legacy system until the old application is no longer needed.

Each capability is migrated independently, reducing risk and enabling continuous delivery of new functionality.

Advantages

  • Reduces migration risk
  • Allows gradual modernization
  • Minimizes downtime
  • Delivers business value continuously

Challenges

  • Temporary system complexity
  • Requires routing and integration logic
  • Data synchronization concerns
  • Long migration timelines for large systems

Best Practices

  • Start with low-risk, independent features.
  • Use an API Gateway or proxy for request routing.
  • Monitor both old and new systems during migration.
  • Automate testing to validate behavior consistency.
  • Retire legacy components as soon as replacement is stable.
The Strangler Fig Pattern enables safe, incremental modernization of legacy applications while keeping the business running.
