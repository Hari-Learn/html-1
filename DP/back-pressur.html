🌊 Backpressure Pattern

The Backpressure pattern controls the flow of data between a producer and a consumer. If the consumer cannot process data fast enough, the producer slows down or pauses to prevent overload.

How It Works

  1. The producer generates data.
  2. The consumer processes the data.
  3. If the consumer becomes busy, it signals the producer.
  4. The producer slows down or waits before sending more data.

Pseudo Code

while (producer.hasData()) {

    if (consumer.isReady()) {
        consumer.process(data);
    } else {
        wait();
    }

}

Commonly used in reactive streams, Apache Kafka, Apache Flink, message queues, and asynchronous systems to maintain stable throughput and prevent resource exhaustion.
