Active Record Pattern

An integration pattern where an object represents both application data and the logic required to store, retrieve, update, and delete that data from a database.

Overview

In the Active Record pattern, database tables are mapped directly to classes, and rows are mapped to objects. Each object contains both business data and persistence methods, allowing it to interact directly with the database.

Key Components

Active Record Object

Represents a database row and contains persistence methods.

Business Data

Stores application-specific fields such as name, email, or status.

Database

Persists and retrieves object data from tables.

Workflow

Object Created
➡️
save()
➡️
Database
➡️
find() / update() / delete()

Example

A User class directly manages its own persistence: 

class User {
  id;
  name;
  email;

  save();
  update();
  delete();
  find();
}

The object contains both the data and the database operations.

Advantages

  • Simple and easy to understand.
  • Reduces boilerplate code.
  • Speeds up CRUD application development.
  • Works well for small and medium-sized applications.
  • Common in many ORM frameworks.
