Request-Reply Design Pattern

A communication pattern where a client sends a request and waits for a response from a service.

Overview

The Request-Reply pattern is commonly used in distributed systems, APIs, microservices, and client-server applications. The client initiates a request, the server processes it, and then sends back a reply containing the result.

Flow

Client
➜ Request ➜
Server
➜ Reply ➜
Response

Real-World Example

Scenario: A user logs into an application.

  • User submits username and password.
  • Application sends a request to the authentication service.
  • Authentication service validates credentials.
  • Service returns a success or failure response.

Advantages

Simple

Easy to understand and implement.

Reliable

Clients receive confirmation through replies.

Widely Used

Common in REST APIs, RPC, and microservices.
