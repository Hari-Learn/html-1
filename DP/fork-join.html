Fork-Join Pattern

The Fork-Join pattern is a concurrency pattern used to divide a large task into smaller independent subtasks (Fork), execute them in parallel, and then combine their results (Join) into a final result.

Workflow

Large Task
↓ Fork
Task A
Task B
Task C
↓ Join
Combined Result

Real-World Example

Imagine calculating the total sales of a company. Instead of processing all records sequentially, the data is split into multiple chunks and processed by different threads simultaneously. Once all threads finish, their results are merged to produce the final total sales amount.

Benefits

  • Improves performance using parallel execution.
  • Efficiently utilizes multi-core processors.
  • Reduces execution time for large computations.
  • Scales well for data-intensive workloads.
// JavaScript Example using Promise.all()

async function processTasks() {
  const taskA = Promise.resolve(10);
  const taskB = Promise.resolve(20);
  const taskC = Promise.resolve(30);

  const results = await Promise.all([
    taskA,
    taskB,
    taskC
  ]);

  const total = results.reduce(
    (sum, value) => sum + value,
    0
  );

  console.log(total); // 60
}

processTasks();
