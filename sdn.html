Software-Defined Networking (SDN)

SDN separates network control from hardware to allow centralized and programmable network management.

In SDN architecture, the network is controlled by a centralized controller instead of individual devices. Network administrators can manage and automate the network more easily using software and APIs.

Important SDN Concepts

Control Plane

The control plane makes network decisions. In SDN, it is centralized in the SDN controller instead of being inside each network device.

Data Plane

The data plane forwards packets through the network. It follows instructions from the SDN controller.

Overlay / Underlay Networks

The underlay network is the physical infrastructure. The overlay network is a virtual network built on top of the underlay.

Northbound APIs

Northbound APIs allow communication between applications and the SDN controller.

Southbound APIs

Southbound APIs connect the SDN controller to network devices such as switches and routers.
