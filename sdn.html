Software-Defined Networking (SDN)

Understanding the SDN architecture and its core networking layers

Software-Defined Networking separates the network control logic from hardware devices. A centralized controller manages the network and communicates with devices using APIs.

Important SDN Concepts

Control Plane

The control plane decides how packets move through the network. In SDN this logic is centralized in the SDN controller.

Data Plane

The data plane is responsible for forwarding packets between devices based on rules from the controller.

Overlay / Underlay Networks

The underlay network is the physical infrastructure. The overlay network is a virtual network running on top of the physical network.

Northbound APIs

Northbound APIs allow applications and management systems to communicate with the SDN controller.

Southbound APIs

Southbound APIs connect the SDN controller with network devices like routers and switches.
