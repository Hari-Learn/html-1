PAT (Port Address Translation) – CCNA

What is PAT?

PAT (Port Address Translation), also called NAT Overload, allows multiple private IP addresses inside a network to share one public IP address by using different port numbers.

What Problem Does PAT Solve?

  • IPv4 public IP addresses are limited
  • Organizations and homes have many devices
  • Each device needs internet access
  • Giving every device a public IP is not possible

Why Does PAT Exist?

PAT exists to allow many devices to use the internet at the same time while using only one public IP address.

The router keeps track of connections using source port numbers.

PAT and TCP 3-Way Handshake (No Duplicate SYN-ACK)

During the TCP 3-way handshake:

  1. Client sends SYN
  2. Server replies with SYN-ACK
  3. Client sends ACK

If multiple internal devices send SYN packets using the same public IP, the router must ensure that the returning SYN-ACK packets go to the correct device.

How PAT prevents duplicate return packets:

  • Each internal connection is assigned a unique source port
  • The router stores this in a NAT table
  • When SYN-ACK returns, the router checks the port number
  • The packet is sent to the correct private IP

Example

Device Private IP Public IP Port
Laptop 192.168.1.10 203.0.113.5 30001
Phone 192.168.1.11 203.0.113.5 30002

Summary

  • PAT allows many devices to share one public IP
  • It solves IPv4 address shortage
  • Uses port numbers to track connections
  • Ensures correct SYN-ACK delivery in TCP handshake
