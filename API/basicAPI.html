🌐 API Basics (Page 1)

🍽️ What is an API?

Imagine you're at a restaurant.

You tell the waiter what you want 🍕, and the waiter brings it from the kitchen.

👉 The waiter = API
👉 The kitchen = server

🚪 What is an Endpoint?

Think of a big building with many doors.

Each door 🚪 leads to something different:

  • One door → user info 👤
  • Another door → photos 🖼️

👉 Each door is an endpoint

📬 HTTP Methods (Actions)

These are like buttons telling what you want to do:

  • 📖 GET → Read something
  • 📤 POST → Send something new
  • ✏️ PUT → Update something
  • 🗑️ DELETE → Remove something

📤 What is a Request?

A request is when you ask for something.

Like saying: "I want a burger 🍔"

👉 You → send a request to the server

📥 What is a Response?

A response is the answer you get back.

Like receiving your burger 🍔 from the waiter.

👉 Server → sends a response back to you

🎯 Simple idea: You ask → API helps → You get results!
