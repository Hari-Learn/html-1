PortFast, CDP, and LLDP Explained

1. PortFast

PortFast is a Cisco Spanning Tree feature that allows a switch port to move immediately into the forwarding state.

Why PortFast is used

  • Speeds up device connectivity (PCs, printers, IP phones)
  • Avoids 30â€“50 seconds of STP delay
  • Improves user experience

What a Network Engineer should know

  • Use only on access ports, not switch-to-switch links
  • Misuse can cause loops
  • Often combined with BPDU Guard for safety

2. CDP (Cisco Discovery Protocol)

CDP is a Cisco proprietary Layer 2 protocol used to discover directly connected Cisco devices.

Why CDP is used

  • Identifies neighboring Cisco devices
  • Shows device type, IP, interface, and OS
  • Helps with troubleshooting and documentation

What a Network Engineer should know

  • Works only with Cisco devices
  • Enabled by default on Cisco switches
  • Should be disabled on untrusted or public-facing ports

3. LLDP (Link Layer Discovery Protocol)

LLDP is an open-standard Layer 2 discovery protocol defined by IEEE 802.1AB.

Why LLDP is used

  • Works across multi-vendor environments
  • Helps identify neighbors and topology
  • Commonly used with IP phones and wireless APs

What a Network Engineer should know

  • Vendor-neutral (unlike CDP)
  • Often disabled by default
  • LLDP-MED provides voice and QoS info
