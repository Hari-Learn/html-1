How Tracert / Traceroute Works

Understanding network path discovery step by step

1. What is Tracert?

Tracert (Windows) or Traceroute (Linux/macOS) is a network diagnostic tool used to track the path that packets take from your computer to a destination host. It shows each hop (router) along the route and measures the delay between hops.

Example:

C:\> tracert google.com
1   1 ms   1 ms   2 ms   192.168.1.1
2   10 ms  11 ms  9 ms   10.0.0.1
3   30 ms  29 ms  31 ms  142.250.190.78

2. Core Concept

The core concept behind tracert/traceroute is the TTL (Time To Live) field in the IP header.

  • Each packet has a TTL value.
  • Every router decreases TTL by 1.
  • When TTL reaches 0, the router drops the packet.
  • The router sends back an ICMP "Time Exceeded" message.

3. Protocols Used

ICMP

Used by Windows tracert. Sends ICMP Echo Requests and receives ICMP Time Exceeded replies.

UDP

Default protocol for Linux traceroute. Sends UDP packets to high-numbered ports.

TCP

Optional method. Sends TCP SYN packets (useful when ICMP is blocked).

4. Step-by-Step Process (Visual Timeline)

Step 1: TTL = 1

First router drops packet → sends ICMP Time Exceeded.

Step 2: TTL = 2

Second router drops packet → sends ICMP reply.

Continue Increasing TTL

Process continues until destination responds.

Final Step: Destination Reached

Destination sends ICMP Echo Reply or Port Unreachable.

5. RTT (Round Trip Time) Measurement

RTT is the time it takes for a packet to travel to a router and back. Tracert typically sends 3 probes per hop and displays three RTT values.

RTT = (Time Reply Received) - (Time Packet Sent)

6. What Does * (Asterisk) Mean?

  • Request timed out.
  • Router blocked ICMP responses.
  • Firewall filtering.
  • High latency or packet loss.
4   *   *   *   Request timed out.
