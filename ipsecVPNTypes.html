Types of IPsec VPN

Site-to-Site, Remote Access, and Host-to-Host VPN Explained

Overview

IPsec VPN can be deployed in different ways depending on business needs. The main types are:

  • Site-to-Site VPN
  • Remote Access VPN
  • Host-to-Host VPN

1. Site-to-Site VPN

What It Is

Site-to-Site VPN connects two or more networks securely over the internet. Usually used between branch offices and headquarters. VPN gateways (routers or firewalls) handle encryption.

When to Use It

  • Connecting branch offices.
  • Linking company to data center or cloud.
  • Secure communication between different business locations.

Example:

Company HQ in New York securely connects to its branch office in London using an IPsec tunnel.

2. Remote Access VPN

What It Is

Remote Access VPN allows individual users to connect securely to a private network from anywhere using the internet. A VPN client is usually installed on the user’s device.

When to Use It

  • Employees working from home.
  • IT administrators managing systems remotely.
  • Secure access while traveling.

Example:

An employee connects from home laptop to company internal servers securely using IPsec VPN client.

3. Host-to-Host VPN

What It Is

Host-to-Host VPN secures communication directly between two individual devices. Encryption is handled by the devices themselves instead of a gateway.

When to Use It

  • Secure communication between two servers.
  • Protecting specific application traffic.
  • High-security environments requiring end-to-end encryption.

Example:

Two database servers encrypt traffic directly between them using IPsec policies.

Quick Comparison

Type Connects Best For
Site-to-Site Network ↔ Network Branch offices
Remote Access User ↔ Network Remote employees
Host-to-Host Device ↔ Device Server-to-server security
