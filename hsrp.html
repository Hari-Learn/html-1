Networking Concepts Explained

HSRP, ISP Failover, Floating Static Routes, Floating IP, and Administrative Distance

What is HSRP?

HSRP (Hot Standby Router Protocol) is a Cisco proprietary redundancy protocol used to provide high availability for the default gateway.

Multiple routers form a group and share a virtual IP address. One router is active, and another is standby. If the active router fails, the standby router takes over automatically.

What is ISP Failover?

ISP Failover ensures continuous internet connectivity by switching traffic from a primary ISP to a backup ISP when the primary link fails.

This is commonly implemented using tracking, SLA monitoring, floating static routes, or dynamic routing protocols.

What is a Floating Static Route?

A floating static route is a static route configured with a higher administrative distance than the primary route.

It remains inactive (floats) until the primary route fails, then it becomes active automatically.

What is a Floating IP?

A Floating IP is a virtual IP address that can move between devices or servers automatically.

It is commonly used in high-availability setups so services remain reachable even if one device fails.

Administrative Distance (AD)

Administrative Distance is a value that represents the trustworthiness of a routing source. Lower values are preferred.

Routers use AD to choose the best route when multiple routing protocols advertise the same destination.

Administrative Distance Values

  • Connected Interface – 0
  • Static Route – 1
  • EIGRP (Internal) – 90
  • OSPF – 110
  • IS-IS – 115
  • RIP – 120
  • EIGRP (External) – 170
  • iBGP – 200
  • Unknown – 255 (Not Used)
