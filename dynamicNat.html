Dynamic NAT Shares a Pool of Public Addresses

Dynamic Network Address Translation allows private hosts to temporarily borrow public IP addresses from a shared pool. These mappings are created when needed and removed when communication ends.

What Is Dynamic NAT?

Dynamic NAT creates a temporary one-to-one mapping between a private IP address and a public IP address selected from a pool.

The mapping exists only while the session is active.

Why It Exists

Not all internal devices need internet access at the same time. Dynamic NAT allows organizations to reuse public IP addresses efficiently.

This reduces the number of public addresses required.

Problem It Solves

Private IP addresses are not routable on the internet. Static NAT requires one public IP per device.

Dynamic NAT solves this by allowing limited public IPs to serve many users over time.

How Dynamic NAT Works

When traffic begins, the router assigns the first available public IP from the pool and records the translation.

Return traffic is matched to this entry and forwarded correctly.

Example Scenario

A company owns ten public IP addresses but has fifty employees. Only active sessions consume public IPs.

When a session ends, the IP is returned to the pool.
