DNS Server and DNS Client – Tutorial

1. What is DNS?

DNS (Domain Name System) is a naming system used on the internet. It converts domain names that humans understand into IP addresses that computers use to communicate.

Example:
www.google.com → 142.250.190.14

2. DNS Server

A DNS server stores DNS records and responds to DNS queries. When a DNS client requests an IP address for a domain, the DNS server provides the correct response.

  • Stores domain-to-IP mappings
  • Resolves domain name queries
  • Works as part of a global DNS hierarchy

3. DNS Client

A DNS client is any device or application that requests DNS resolution. Browsers, operating systems, and servers act as DNS clients.

When you type a website address in a browser, your system becomes a DNS client and sends a query to a DNS server.

4. Why DNS is Required

Computers communicate using IP addresses, not names. DNS allows users to access websites using easy-to-remember names.

  • Simplifies internet usage
  • Reduces dependency on fixed IP addresses
  • Supports scalability of the internet

5. Preferred DNS Server

The preferred DNS server is the primary DNS server configured on a system. All DNS queries are sent to this server first.

Example: 8.8.8.8

6. Alternative DNS Server

The alternative DNS server acts as a backup. If the preferred DNS server fails to respond, the system automatically queries the alternative server.

Example: 8.8.4.4

7. Registering an FQDN in a DNS Server

An FQDN (Fully Qualified Domain Name) uniquely identifies a host on the internet.

  1. Register a domain name with a registrar
  2. Access the DNS management console
  3. Create an A record for the host name
  4. Assign the IP address to the FQDN
  5. Save the configuration
  6. Wait for DNS propagation

Example: www.example.com → 192.168.1.10
