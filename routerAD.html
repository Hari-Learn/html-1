Administrative Distance (AD)

Routing Protocol Preference Explained (CCNA)

What is Administrative Distance?

Administrative Distance (AD) is a value used by routers to determine which routing protocol is more trustworthy when multiple protocols provide routes to the same destination.

Why Administrative Distance is Needed?

  • A router may learn the same network from multiple routing protocols.
  • Metrics are compared only within the same protocol.
  • AD helps the router decide which protocolâ€™s route to install in the routing table.

Default Administrative Distance Values

Route Source Administrative Distance
Connected Interface 0
Static Route 1
EIGRP (Internal) 90
OSPF 110
RIP 120
EIGRP (External) 170
Unknown 255 (Never used)

Example Scenario

A router learns network 10.1.1.0/24 from:

  • OSPF (AD 110)
  • EIGRP (AD 90)

ðŸ‘‰ The router will choose the EIGRP route because it has a lower AD.

CCNA Key Points

  • AD is checked before the routing metric.
  • Lower AD routes are always preferred.
  • Static routes can be configured with higher AD to act as floating static routes.
  • AD is a local value and is not advertised to other routers.
