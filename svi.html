‚ùå The Problem

Layer-2 switches forward frames using MAC addresses, but they cannot use IP addresses.

  • Switch cannot be pinged
  • No remote management (SSH / Telnet)
  • No inter-VLAN routing
  • Each VLAN is isolated forever

So the question is: How can a switch participate in Layer-3 communication?

‚ö†Ô∏è Why Physical Interfaces Are Not Enough

Switch physical ports are Layer-2 only. You cannot assign IP addresses to switch ports like a router. 

Switch Port ‚Üí MAC-based forwarding only
No IP address
No routing

‚úÖ Solution: Switch Virtual Interface (SVI)

An SVI is a virtual Layer-3 interface created for a VLAN on a switch.

What SVI does:

  • Provides an IP address to a VLAN
  • Acts as the default gateway for devices
  • Enables inter-VLAN routing
  • Allows remote switch management
