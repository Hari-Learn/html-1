Overlay vs Underlay Network Virtualization

Understanding Virtual Topology and Physical Connectivity

Network Virtualization

Network virtualization is the process of creating multiple virtual networks on top of a shared physical network infrastructure. It allows organizations to run multiple isolated networks using the same physical hardware.

Technologies like VXLAN, GRE, and SDN controllers help implement virtualization in modern cloud and data center environments.

Underlay Network

The underlay network is the physical infrastructure that connects devices. It includes routers, switches, and cables that provide basic connectivity.

  • Physical routers and switches
  • Provides IP connectivity
  • Uses routing protocols
  • Forms the foundation for overlay networks

Overlay Network

An overlay network is a virtual network built on top of the underlay network. It uses tunneling protocols to connect virtual machines or containers.

  • Runs on top of the physical network
  • Creates logical connections
  • Uses VXLAN or GRE tunnels
  • Provides network isolation

Virtual Topology

Virtual topology describes the logical arrangement of devices within an overlay network. Devices may appear directly connected even though they are located on different physical servers.

This allows flexible network design and easier management in cloud environments.

Physical Connectivity

Physical connectivity refers to the real-world hardware connections between networking devices such as switches, routers, and servers.

  • Ethernet cables or fiber links
  • Switch-to-switch connections
  • Router communication
  • Forms the underlay infrastructure
