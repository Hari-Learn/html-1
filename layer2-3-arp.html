ARP Protocol Behavior

Layer 2 and Layer 3 Interaction

What is ARP?

ARP (Address Resolution Protocol) is used to map an IP address (Layer 3) to a MAC address (Layer 2) so that data can be delivered within a local network.

ARP Behavior at Layer 2 (Data Link Layer)

  • ARP uses MAC addresses for communication.
  • ARP Request is sent as a broadcast frame.
  • Destination MAC for ARP Request: FF:FF:FF:FF:FF:FF.
  • ARP Reply is sent as a unicast frame.
  • Switches forward ARP frames using Layer 2 logic.

ARP Behavior at Layer 3 (Network Layer)

  • ARP works with IP addresses.
  • Checks whether the destination IP is in the same network.
  • If destination is local, ARP finds the destination MAC.
  • If destination is remote, ARP resolves the default gateway's MAC.
  • Routers do not forward ARP broadcasts.

Steps of ARP Working

  1. Host checks ARP cache for destination IP.
  2. If not found, sends ARP Request (broadcast).
  3. Target device sends ARP Reply (unicast).
  4. MAC address is stored in ARP table.
  5. Data transmission begins.
