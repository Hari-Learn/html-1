VTP (VLAN Trunking Protocol) - CCNA

Understanding VTP concepts for Cisco networking

What is VTP?

VLAN Trunking Protocol (VTP) is a Cisco proprietary protocol used to manage VLAN configurations across multiple switches in a network. It helps maintain VLAN consistency by distributing VLAN information automatically.

VTP Modes

  • Server Mode: Can create, modify, and delete VLANs. Changes are advertised to other switches.
  • Client Mode: Cannot create VLANs but receives updates from VTP servers.
  • Transparent Mode: Does not participate in VTP updates but forwards VTP advertisements.

VTP Requirements

  • Same VTP domain name
  • Trunk links between switches
  • Matching VTP version
  • Same VTP password (if configured)

VTP Versions

  • Version 1: Default version, supports normal VLANs.
  • Version 2: Adds support for Token Ring VLANs.
  • Version 3: Supports extended VLANs and better security.

Advantages of VTP

  • Centralized VLAN management
  • Reduces configuration errors
  • Saves time in large networks
