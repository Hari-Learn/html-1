DHCP Client–Server Flow (DORA)

When a device joins a network, it follows the DORA process to obtain an IP address from the DHCP server.

Discover

Client broadcasts a request to find DHCP servers.

Offer

Server offers an available IP address.

Request

Client requests the offered IP.

Acknowledge

Server confirms and assigns the IP.

Flow When DHCP Server Is Unreachable (APIPA)

If no DHCP server responds, the client automatically assigns itself an APIPA address to allow limited local communication.

No Response

Client sends Discover messages but gets no reply.

Self Assignment

Client assigns an IP from 169.254.0.0/16.

Limited Access

Only local network communication is possible.
